Section 1: Java Ecosystem for Automation

Before writing a single line of Selenium or REST Assured code, automation engineers must understand the Java runtime environment that executes test suites. This section maps JDK, JRE, and JVM concepts directly to how your framework compiles, runs, and scales across CI pipelines.

JDK vs JRE vs JVM — Automation Context

JDK (Java Development Kit)

The JDK is what automation engineers install on their machines and CI agents. It contains the compiler (javac), debugger, JAR tools, and the JRE. When you build a Maven or Gradle project, the JDK compiles your Page Objects, TestNG classes, and Cucumber step definitions into bytecode.

JRE (Java Runtime Environment)

The JRE provides the libraries and JVM needed to execute compiled test suites. In production CI, some pipelines only need the JRE to run pre-built JARs — but for development and compilation you always need the full JDK.

JVM (Java Virtual Machine)

The JVM is the engine that runs your compiled test code. It loads .class files, manages memory (heap for WebDriver instances, stack for method calls), and performs garbage collection when test objects are no longer referenced. Every parallel TestNG thread runs inside the same JVM process (unless you fork per suite).

Why Platform Independence Matters: The same compiled test JAR runs on Windows CI (Jenkins), Linux Docker containers (Selenium Grid), and Mac developer laptops without recompilation. This is the foundation of cross-platform automation pipelines.

JVM Memory for Test Execution

Heap Memory

Stores all objects created during test execution: WebDriver instances, Page Object references, test data HashMaps, API response objects, and Cucumber scenario context. When you run 10 parallel TestNG threads, each thread's objects compete for heap space.

Stack Memory

Stores method call frames and local variables. Each TestNG test method gets its own stack frame. When loginTest() calls loginPage.enterUsername(), a new frame is pushed onto the stack.

Method Area (Metaspace in Java 8+)

Stores class metadata, static variables, and constant pool. Your framework's static config (BASE_URL, TIMEOUT) and class definitions for all Page Objects live here.

Why Memory Matters in Automation: Large test suites (500+ tests) with parallel execution (5-10 threads) can cause OutOfMemoryError if WebDriver instances aren't closed in @AfterMethod. Always call driver.quit() — not just close() — to release browser processes and heap memory.

Garbage Collection Basics

Garbage Collection (GC) automatically reclaims heap memory from objects no longer referenced. When a test method finishes and the WebDriver reference goes out of scope, GC marks that object for cleanup.

  • Automatic cleanup — No manual free() like C/C++. JVM handles it.
  • System.gc() — Suggests GC run; never call in test code — JVM decides optimal timing.
  • finalize() — Deprecated since Java 9. Do not rely on it for driver cleanup.

When GC matters in automation:

  • Long-running test suites (regression suites running 2+ hours)
  • Parallel execution creating hundreds of WebDriver objects
  • Memory leaks from unclosed drivers or unbounded Collections in test data caches
  • Selenium Grid nodes running multiple sessions sequentially

Interview Q&A — Java Ecosystem for Automation

Q1: What is the difference between JDK, JRE, and JVM?

A: JDK is the full development kit (compiler + tools + JRE). JRE is the runtime environment (JVM + libraries) for executing programs. JVM is the virtual machine that runs bytecode. Automation engineers install JDK to compile and run tests.

Q2: Why is Java platform-independent and why does it matter for test automation?

A: Java source compiles to platform-independent bytecode (.class), which any JVM runs regardless of OS. This means the same test JAR executes on Windows Jenkins agents, Linux Docker containers, and Mac laptops — critical for CI/CD pipelines.

Q3: What happens when you run mvn test?

A: Maven compiles source code (javac), compiles test code, resolves dependencies from pom.xml, then invokes the Surefire plugin which uses TestNG/JUnit to load compiled test classes into the JVM and execute them.

Q4: What is bytecode?

A: Intermediate code generated by javac that the JVM interprets and JIT-compiles to native code. Your LoginPage.java becomes LoginPage.class — a bytecode file the JVM understands on any platform.

Q5: Explain heap vs stack memory with an automation example.

A: Heap stores objects like WebDriver driver = new ChromeDriver() — shared across threads. Stack stores local variables like String username = "admin" inside a test method — private to that thread's method call.

Q6: What causes OutOfMemoryError in test automation?

A: Not closing WebDriver instances (driver.quit()), running too many parallel threads without increasing heap (-Xmx), caching unlimited test data in static Collections, or large screenshot attachments stored in memory.

Q7: What is Garbage Collection and when should you think about it?

A: GC automatically frees heap memory from unreferenced objects. Think about it when running long regression suites, parallel tests, or when noticing increasing memory usage across test runs — usually indicates a driver leak.

Q8: Should you call System.gc() in your test framework?

A: No. It's a suggestion, not a command. The JVM manages GC optimally. Instead, ensure proper cleanup in @AfterMethod — close drivers, clear ThreadLocal, null out large objects.

Q9: Which JDK version should a new automation project use?

A: JDK 17 (current LTS) for new projects. It supports modern Selenium 4.x, records for test data, and has long-term support. Verify all dependencies (Selenium, TestNG, REST Assured) support the chosen version.

Q10: What is the role of Maven in an automation framework?

A: Dependency management (Selenium, TestNG versions), build lifecycle (compile → test → package), plugin execution (Surefire for tests, Allure for reporting), and standard project structure (src/main, src/test).

Q11: What is JIT compilation and how does it help test execution?

A: Just-In-Time compiler converts frequently executed bytecode to native machine code at runtime. In automation, repeated calls to utility methods (waits, element finding) get JIT-optimized, making later test runs faster within the same JVM session.

Q12: What is the Method Area / Metaspace?

A: Stores class-level data: class metadata, static variables, constant pool. Your framework's static String BASE_URL and all Page Object class definitions reside here. Metaspace replaced PermGen in Java 8+.

Q13: How do you configure JVM memory for parallel test execution?

A: Use -Xmx2g -Xms1g in Maven Surefire plugin or TestNG XML. Increase heap when running 5+ parallel threads. Example: mvn test -Dsurefire.argLine="-Xmx2g".

Q14: What is the difference between compile-time and runtime in Java automation?

A: Compile-time: javac checks types, resolves imports, catches syntax errors before tests run. Runtime: JVM executes tests, WebDriver launches browsers, assertions run — errors here are test failures or exceptions.

Q15: Can you run tests with only JRE installed?

A: Yes, if tests are pre-compiled into a JAR (e.g., CI pipeline with separate build and execution stages). But developers always need JDK for writing and compiling new test code.

Q16: What is a .jar file in automation context?

A: Java Archive — a packaged collection of compiled .class files and resources. Your framework JAR contains all Page Objects and utilities. Test JAR contains test classes. Maven produces these in the target/ directory.

Section 2: OOPS Concepts — The Foundation of Every Framework

Every Selenium Page Object Model, TestNG base class, Cucumber hook, and REST Assured wrapper is built on Object-Oriented Programming. Master these four pillars — Encapsulation, Inheritance, Polymorphism, and Abstraction — and you understand 90% of any automation framework's architecture.

Class & Object

A class is a blueprint that defines properties (fields) and behaviors (methods). An object is an instance of that class created at runtime.

  • Class = blueprint for Page Objects, Test classes, Utility classes
  • Object = instance: LoginPage loginPage = new LoginPage(driver);
public class LoginPage {
    private WebDriver driver;
    private By usernameField = By.id("username");
    private By passwordField = By.id("password");
    private By loginButton   = By.id("login-btn");

    public LoginPage(WebDriver driver) {
        this.driver = driver;
    }

    public void enterUsername(String username) {
        driver.findElement(usernameField).sendKeys(username);
    }

    public void enterPassword(String password) {
        driver.findElement(passwordField).sendKeys(password);
    }

    public void clickLogin() {
        driver.findElement(loginButton).click();
    }

    public void login(String user, String pass) {
        enterUsername(user);
        enterPassword(pass);
        clickLogin();
    }
}

// Usage — creating objects
LoginPage loginPage = new LoginPage(driver);
loginPage.login("admin", "password123");

Constructor

Constructors initialize objects when new is called. They share the class name and have no return type.

Default Constructor

Java provides a no-argument constructor if you define none. Rarely used in automation — most classes need dependencies injected.

Parameterized Constructor

Accepts arguments to initialize state. Standard pattern for Page Objects receiving WebDriver.

Constructor Chaining

this() calls another constructor in the same class. super() calls the parent class constructor — required as the first line when extending a base class.

Encapsulation

Encapsulation = data hiding. Private fields accessed only through public methods.

public class ConfigReader {
    private Properties properties;

    public ConfigReader(String filePath) {
        properties = new Properties();
        try (InputStream input = new FileInputStream(filePath)) {
            properties.load(input);
        } catch (IOException e) {
            throw new RuntimeException("Failed to load config: " + filePath, e);
        }
    }

    public String getProperty(String key) {
        return properties.getProperty(key);
    }

    public String getBaseUrl() {
        return properties.getProperty("base.url");
    }

    public int getTimeout() {
        return Integer.parseInt(properties.getProperty("timeout", "30"));
    }
}

public class DriverManager {
    private static ThreadLocal<WebDriver> driver = new ThreadLocal<>();

    private DriverManager() {}

    public static WebDriver getDriver() {
        if (driver.get() == null) {
            throw new IllegalStateException("Driver not initialized.");
        }
        return driver.get();
    }

    public static void setDriver(WebDriver webDriver) {
        driver.set(webDriver);
    }

    public static void quitDriver() {
        if (driver.get() != null) {
            driver.get().quit();
            driver.remove();
        }
    }
}

Inheritance

Inheritance creates an IS-A relationship using extends.

Types of Inheritance in Automation Frameworks

  • Single: LoginTest extends BaseTest
  • Multilevel: BaseTest → WebTestBase → LoginTest
  • Hierarchical: BasePage → LoginPage, DashboardPage, SettingsPage
public class BaseTest {
    protected WebDriver driver;
    protected ConfigReader config;

    @BeforeMethod
    public void setUp() {
        config = new ConfigReader("config.properties");
        driver = DriverFactory.createDriver(config.getProperty("browser"));
        DriverManager.setDriver(driver);
        driver.get(config.getBaseUrl());
    }

    @AfterMethod
    public void tearDown() {
        DriverManager.quitDriver();
    }
}

public class LoginTest extends BaseTest {
    @Test
    public void testValidLogin() {
        LoginPage loginPage = new LoginPage(driver);
        loginPage.login("admin", "password123");
        Assert.assertTrue(driver.getCurrentUrl().contains("dashboard"));
    }

    @Test
    public void testInvalidLogin() {
        LoginPage loginPage = new LoginPage(driver);
        loginPage.login("invalid", "wrong");
        Assert.assertTrue(loginPage.isErrorDisplayed());
    }
}

public class BasePage {
    protected WebDriver driver;
    protected WebDriverWait wait;

    public BasePage(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(30));
    }

    public void click(By locator) {
        wait.until(ExpectedConditions.elementToBeClickable(locator)).click();
    }

    public String getText(By locator) {
        return wait.until(ExpectedConditions.visibilityOfElementLocated(locator)).getText();
    }

    public boolean isDisplayed(By locator) {
        try {
            return driver.findElement(locator).isDisplayed();
        } catch (NoSuchElementException e) {
            return false;
        }
    }
}

public class LoginPage extends BasePage {
    private By usernameField = By.id("username");
    private By passwordField = By.id("password");
    private By loginButton   = By.id("login-btn");
    private By errorMessage  = By.cssSelector(".error-msg");

    public LoginPage(WebDriver driver) {
        super(driver);
    }

    public void login(String user, String pass) {
        driver.findElement(usernameField).sendKeys(user);
        driver.findElement(passwordField).sendKeys(pass);
        click(loginButton);
    }

    public boolean isErrorDisplayed() {
        return isDisplayed(errorMessage);
    }
}
Why Multiple Inheritance is Not Supported: Java classes can extend only one parent (diamond problem). Use interfaces instead — e.g., extends BasePage implements IPageActions, IScreenshotCapable.

Polymorphism

Compile-Time Polymorphism (Method Overloading)

public class BasePage {
    protected WebDriver driver;

    public BasePage(WebDriver driver) { this.driver = driver; }

    public void click(By locator) {
        driver.findElement(locator).click();
    }

    public void click(By locator, int timeoutSeconds) {
        WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(timeoutSeconds));
        wait.until(ExpectedConditions.elementToBeClickable(locator)).click();
    }

    public void click(WebElement element) {
        element.click();
    }

    public void click(WebElement element, int timeoutSeconds) {
        WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(timeoutSeconds));
        wait.until(ExpectedConditions.elementToBeClickable(element)).click();
    }
}

Runtime Polymorphism (Method Overriding)

The most important concept in Selenium: WebDriver driver = new ChromeDriver();

WebDriver driver = new ChromeDriver();
WebDriver driver = new FirefoxDriver();
WebDriver driver = new EdgeDriver();

public class DriverFactory {
    public static WebDriver createDriver(String browserType) {
        WebDriver driver;
        switch (browserType.toLowerCase()) {
            case "chrome":  driver = new ChromeDriver(); break;
            case "firefox": driver = new FirefoxDriver(); break;
            case "edge":    driver = new EdgeDriver(); break;
            default: throw new IllegalArgumentException("Unsupported: " + browserType);
        }
        driver.manage().window().maximize();
        return driver;
    }
}

    protected WebDriverWait wait;

2. Method Overloading — search()

public class TestDataSearch {
    private List<Employee> employees;
    public TestDataSearch(List<Employee> employees) { this.employees = employees; }

    public Employee search(String name) {
        return employees.stream().filter(e -> e.getName().equalsIgnoreCase(name))
            .findFirst().orElse(null);
    }
    public Employee search(int id) {
        return employees.stream().filter(e -> e.getId() == id)
            .findFirst().orElse(null);
    }
    public Employee search(String name, String department) {
        return employees.stream()
            .filter(e -> e.getName().equalsIgnoreCase(name)
                      && e.getDepartment().equalsIgnoreCase(department))
            .findFirst().orElse(null);
    }
}

3. Method Overriding — Shape Hierarchy

abstract class Shape {
    abstract double area();
    abstract String getShapeName();
}
class Circle extends Shape {
    private double radius;
    public Circle(double radius) { this.radius = radius; }
    @Override double area() { return Math.PI * radius * radius; }
    @Override String getShapeName() { return "Circle"; }
}
class Rectangle extends Shape {
    private double width, height;
    public Rectangle(double w, double h) { width = w; height = h; }
    @Override double area() { return width * height; }
    @Override String getShapeName() { return "Rectangle"; }
}
// Output: Circle area: 78.54 | Rectangle area: 24.0

Interview Q&A — OOPS (4+ Years)

Q1: Explain OOPS pillars with automation examples.

A: Encapsulation: ConfigReader hides Properties. Inheritance: LoginTest extends BaseTest. Polymorphism: WebDriver driver = new ChromeDriver(). Abstraction: abstract BasePage with verifyPageLoaded().

Q2: Overloading vs Overriding?

A: Overloading: same name, different params, compile-time (click(By) vs click(By, int)). Overriding: same signature, parent-child, runtime (@Override verifyPageLoaded).

Q3: Why is WebDriver an interface?

A: Polymorphism — test code uses WebDriver reference while runtime gets ChromeDriver/FirefoxDriver. Enables browser factory and Grid compatibility.

Q4: Abstract class vs interface in frameworks?

A: Abstract class for BasePage (shared code + abstract methods). Interface for capabilities (IScreenshotCapable) allowing multiple implementation.

Q5: Why no multiple inheritance?

A: Diamond problem. Java allows multiple interface implementation instead.

Q6: Constructor chaining in Page Objects?

A: AdminLoginPage(WebDriver) calls super(driver) to initialize BasePage's driver and wait fields.

Q7: IS-A vs HAS-A?

A: IS-A = inheritance (LoginPage IS-A BasePage). HAS-A = composition (LoginPage HAS-A WebDriver). Favor composition over inheritance.

Q8: Access modifiers in Page Objects?

A: private locators, protected driver/wait, public action methods. Never public locators.

Q9: Polymorphism in cross-browser testing?

A: DriverFactory returns WebDriver. Change browser via config — tests unchanged.

Q10: super keyword usage?

A: super() calls parent constructor; super.methodName() calls parent method.

Q11: Can abstract class have constructor?

A: Yes. BasePage constructor initializes driver/wait; children call via super(driver).

Q12: Method hiding vs overriding?

A: Overriding: instance methods, runtime. Hiding: static methods, compile-time.

Q13: POM using OOPS?

A: Each page = class. extends BasePage. abstract verifyPageLoaded. WebDriver interface refs.

Q14: Coupling vs cohesion?

A: High cohesion: one page per class. Low coupling: tests use page methods, not locators.

Q15: Runtime polymorphism without Selenium?

A: PaymentProcessor p = new UPIPayment(); p.processPayment(100) — JVM resolves at runtime.

Q16: Composition vs inheritance?

A: Inheritance = IS-A, tight coupling. Composition = HAS-A, loose coupling. Prefer composition when not truly IS-A.

Q17: Abstraction without abstract classes?

A: Yes, via interfaces. WebDriver is pure abstraction.

Interview Q&A — Variables & Data Types

Q1: How many primitive types does Java have?

A: 8: byte, short, int, long, float, double, char, boolean. Everything else is an object/reference type.

Q2: What is autoboxing?

A: Automatic conversion of primitive to wrapper — int to Integer. Enables storing primitives in Collections like List<Integer> used in DataProviders.

Q3: Why does Integer == Integer fail for values above 127?

A: Java caches Integer objects from -128 to 127. Above that, == compares references (different objects), not values. Always use .equals() or .intValue().

Q4: Difference between instance and static variables in a Page Object?

A: Instance: each page object has its own driver reference. Static: shared config like BASE_URL — same value for all page objects and tests.

Q5: When do you use explicit casting in Selenium?

A: When calling interface-specific methods: JavascriptExecutor for JS clicks, TakesScreenshot for screenshots. WebDriver is an interface — casting accesses implementation-specific APIs.

Q6: What is the var keyword and when to use it?

A: Local variable type inference (Java 10+). Use for verbose types like Map<String, List<WebElement>>. Avoid for unclear types — explicit types improve readability in test code.

Q7: What is the default value of an uninitialized instance variable?

A: 0 for numeric types, false for boolean, null for object references. Local variables have NO default — compiler error if used uninitialized.

Q8: Can you change a static final variable?

A: No. final means assign once. static final = constant. BASE_URL should never change during test execution.

Q9: Widening vs narrowing casting?

A: Widening (int→long→double) is automatic and safe. Narrowing (double→int) requires explicit cast and may lose data (truncates decimal).

Q10: Why use long for timestamps in performance tests?

A: System.currentTimeMillis() returns long (milliseconds since epoch). int would overflow. Store start/end times as long, calculate difference.

Q11: What happens with unboxing a null Integer?

A: NullPointerException. If API response returns null for a numeric field, check for null before unboxing: if (count != null) int val = count;

Q12: Local vs instance variable — interview example?

A: In testLogin(): String username = "admin" is local (destroyed after method). In LoginPage: private WebDriver driver is instance (lives as long as the page object).

Section 4: Strings — Most Asked in Coding Rounds

String Fundamentals

Immutability

Once created, a String object cannot be changed. Any "modification" creates a new String object in memory.

String locator = "//input[@id='username']";
locator = locator.replace("username", "email");  // creates NEW String, original unchanged

String Pool

JVM maintains a pool of String literals in heap memory. When you create a literal, JVM checks the pool first — if it exists, the reference is reused.

  STACK                          HEAP
  ┌──────────┐                  ┌─────────────────────────────┐
  │  s1 ─────┼─────────────────►│  STRING POOL                │
  │  s2 ─────┼──┐               │  ┌─────────┐               │
  │  s3 ─────┼──┼──────────────►│  │ "hello" │ ◄── s1, s2    │
  └──────────┘  │               │  └─────────┘               │
                │               │                             │
                │               │  HEAP (non-pool)            │

                                │  └─────────┘               │
                                └─────────────────────────────┘

  String s2 = "hello";              // pool — reuses same object

Coding Questions with Complete Solutions

1. Reverse a String

Problem: Reverse a string — common in coding rounds and useful for palindrome checks in test validation.

Approach 1 — StringBuilder (Optimized):

public static String reverse(String str) {
    if (str == null || str.isEmpty()) return str;
    return new StringBuilder(str).reverse().toString();
}
// Time: O(n), Space: O(n)

Approach 2 — Manual Char Swap (Brute Force):

public static String reverseManual(String str) {
    char[] chars = str.toCharArray();
    int left = 0, right = chars.length - 1;
    while (left < right) {
        char temp = chars[left];
        chars[left] = chars[right];
        chars[right] = temp;
        left++; right--;
    }
    return new String(chars);
}
// reverse("hello") → "olleh"

2. Count Vowels

Problem: Count vowels in a string — models input validation testing.

Approach 1 — Loop + indexOf:

public static int countVowels(String str) {
    if (str == null) return 0;
    int count = 0;
    String vowels = "aeiouAEIOU";
    for (char c : str.toCharArray()) {
        if (vowels.indexOf(c) != -1) count++;
    }
    return count;
}

Approach 2 — Stream (Optimized):

public static int countVowelsStream(String str) {
    if (str == null) return 0;
    return (int) str.chars()
        .filter(c -> "aeiouAEIOU".indexOf(c) != -1)
        .count();
}
// countVowels("Automation") → 5

3. Count Words

Problem: Count words in a string — useful for validating text field input limits.

Approach 1 — split (Optimized):

public static int countWords(String str) {
    if (str == null || str.isBlank()) return 0;
    return str.trim().split("\\s+").length;
}

Approach 2 — Manual iteration (Brute Force):

public static int countWordsManual(String str) {
    if (str == null || str.isBlank()) return 0;
    int count = 0;
    boolean inWord = false;
    for (char c : str.trim().toCharArray()) {
        if (!Character.isWhitespace(c) && !inWord) { count++; inWord = true; }
        else if (Character.isWhitespace(c)) { inWord = false; }
    }
    return count;
}
// countWords("Hello World from Selenium") → 4

4. Remove Duplicate Characters

Approach 1 — LinkedHashSet (Optimized):

public static String removeDuplicates(String str) {
    if (str == null) return null;
    StringBuilder sb = new StringBuilder();
    Set<Character> seen = new LinkedHashSet<>();
    for (char c : str.toCharArray()) seen.add(c);
    seen.forEach(c -> sb.append(c));
    return sb.toString();
}

Approach 2 — Stream:

public static String removeDuplicatesStream(String str) {
    return str.chars().distinct()
        .mapToObj(c -> String.valueOf((char) c))
        .collect(Collectors.joining());
}
// removeDuplicates("programming") → "progamin"

5. Find Duplicate Words

public static Set<String> findDuplicateWords(String sentence) {
    Set<String> seen = new HashSet<>();
    Set<String> duplicates = new HashSet<>();
    for (String word : sentence.toLowerCase().split("\\s+")) {
        if (!seen.add(word)) duplicates.add(word);
    }
    return duplicates;
}

6. Palindrome Check

Approach 1 — Reverse Compare (Optimized):

public static boolean isPalindrome(String str) {
    if (str == null) return false;
    String cleaned = str.replaceAll("[^a-zA-Z0-9]", "").toLowerCase();
    return cleaned.equals(new StringBuilder(cleaned).reverse().toString());
}

Approach 2 — Two Pointer:

public static boolean isPalindromeTwoPointer(String str) {
    if (str == null) return false;
    String s = str.replaceAll("[^a-zA-Z0-9]", "").toLowerCase();
    int left = 0, right = s.length() - 1;
    while (left < right) {
        if (s.charAt(left) != s.charAt(right)) return false;
        left++; right--;
    }
    return true;
}
// isPalindrome("A man a plan a canal Panama") → true

7. Anagram Check

Approach 1 — Sort + Compare (Optimized):

public static boolean isAnagram(String s1, String s2) {
    if (s1 == null || s2 == null) return false;
    char[] a = s1.replaceAll("\\s", "").toLowerCase().toCharArray();
    char[] b = s2.replaceAll("\\s", "").toLowerCase().toCharArray();
    if (a.length != b.length) return false;
    Arrays.sort(a); Arrays.sort(b);
    return Arrays.equals(a, b);
}

Approach 2 — Character Frequency:

public static boolean isAnagramFrequency(String s1, String s2) {
    if (s1.length() != s2.length()) return false;
    int[] freq = new int[26];
    for (char c : s1.toLowerCase().toCharArray()) freq[c - 'a']++;
    for (char c : s2.toLowerCase().toCharArray()) if (--freq[c - 'a'] < 0) return false;
    return true;
}
// isAnagram("listen", "silent") → true

8. Character Frequency

Approach 1 — HashMap:

public static Map<Character, Integer> charFrequency(String str) {
    Map<Character, Integer> freq = new HashMap<>();
    for (char c : str.toCharArray()) {
        freq.merge(c, 1, Integer::sum);
    }
    return freq;
}

Approach 2 — Java 8 Collectors.groupingBy (Optimized):

public static Map<Character, Long> charFrequencyStream(String str) {
    return str.chars()
        .mapToObj(c -> (char) c)
        .collect(Collectors.groupingBy(c -> c, Collectors.counting()));
}
// charFrequency("hello") → {h=1, e=1, l=2, o=1}

Interview Q&A — Strings

Q1: Why is String immutable?

A: Security (credentials can't be modified), thread safety (shared across threads), String pool optimization (reuse literals), and hashcode caching. In automation: locator strings shared safely across parallel threads.

Q2: How many objects created: String s = new String("hello")?

A: Two — one "hello" in String pool (if not already there) and one new String object on heap. s references the heap object.

Q3: == vs equals() for Strings?

A: == compares references (memory address). equals() compares content. For assertions and text validation, ALWAYS use equals().

Q4: String vs StringBuilder performance?

A: String concatenation in loops creates N new objects (O(n²)). StringBuilder appends in-place (O(n)). Use StringBuilder for building dynamic XPath in loops.

Q5: What is String pool?

A: Special memory area in heap storing String literals. Reuses existing literals instead of creating duplicates. Literals created with "..." go to pool; new String("...") goes to heap.

Q6: When to use StringBuffer over StringBuilder?

A: When multiple threads modify the same builder — e.g., parallel test report generation where multiple TestNG threads append results. Otherwise, always prefer StringBuilder (faster).

Q7: How do you build dynamic XPath in Selenium?

A: String.format("//input[@id='%s']", elementId) or xpath.replace("PLACEHOLDER", value). Never concatenate with + in loops — use StringBuilder.

Q8: What does intern() do?

A: Moves a heap String to the pool and returns the pool reference. new String("hello").intern() == "hello" → true. Rarely needed in automation.

Q9: isEmpty() vs isBlank()?

A: isEmpty() checks length == 0. isBlank() (Java 11+) checks length == 0 OR all whitespace. Use isBlank() for validating user input fields that may contain spaces.

Q10: How to compare strings ignoring case?

A: equalsIgnoreCase(). Or toLowerCase().equals() — but equalsIgnoreCase handles Unicode correctly and avoids creating new strings.

Q11: split() regex pitfalls?

A: split(".") splits every character (dot is regex wildcard). Use split("\\.") for literal dot. split("|") splits every character. Always escape regex special chars.

Q12: How to validate email format in tests?

A: email.matches("^[\\w.-]+@[\\w.-]+\\.[a-zA-Z]{2,}$"). Or use Apache Commons Validator. Prefer regex matches() for inline test validation.

Q13: concat() vs + operator?

A: Both create new String objects. + is syntactic sugar for StringBuilder.append() (compiler optimization). Use + for simple concatenation, StringBuilder for loops.

Q14: How to extract JSON field without library?

A: String body = response.asString(); then body.contains("\"name\":\"John\"") or split/indexOf for simple cases. For complex JSON, use jsonPath() in REST Assured.

Q15: What happens when you modify a String?

A: Original is unchanged (immutable). A new String object is created. The reference must be reassigned: s = s.replace("old", "new"). Old object becomes eligible for GC.

Coding Questions with Complete Solutions

1. Find Largest Element

Problem: Find the largest number in an array — models finding max response time from performance test results.

Approach — Iterate with Max Tracking (Optimized):

public static int findLargest(int[] arr) {
    if (arr == null || arr.length == 0) throw new IllegalArgumentException("Empty array");
    int max = arr[0];
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] > max) max = arr[i];
    }
    return max;
}
// findLargest({3, 7, 1, 9, 4}) → 9 | Time: O(n), Space: O(1)

2. Find Second Largest

Approach — Single Pass with Two Variables (Optimized):

public static int findSecondLargest(int[] arr) {
    if (arr.length < 2) throw new IllegalArgumentException("Need at least 2 elements");
    int largest = Integer.MIN_VALUE, secondLargest = Integer.MIN_VALUE;
    for (int num : arr) {
        if (num > largest) {
            secondLargest = largest;
            largest = num;
        } else if (num > secondLargest && num != largest) {
            secondLargest = num;
        }
    }
    return secondLargest;
}
// findSecondLargest({3, 7, 1, 9, 4}) → 7 | Time: O(n), Space: O(1)

3. Sort Array

Approach 1 — Arrays.sort() (Optimized):

public static void sortArray(int[] arr) {
    Arrays.sort(arr);  // dual-pivot quicksort — O(n log n)
}
// sortArray({5, 2, 8, 1, 9}) → [1, 2, 5, 8, 9]

Approach 2 — Bubble Sort (Brute Force):

public static void bubbleSort(int[] arr) {
    for (int i = 0; i < arr.length - 1; i++) {
        for (int j = 0; j < arr.length - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j]; arr[j] = arr[j + 1]; arr[j + 1] = temp;
            }
        }
    }
}
// Time: O(n²), Space: O(1) — use only when asked in interviews

Approach 3 — Selection Sort:

public static void selectionSort(int[] arr) {
    for (int i = 0; i < arr.length - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < arr.length; j++) {
            if (arr[j] < arr[minIdx]) minIdx = j;
        }
        int temp = arr[i]; arr[i] = arr[minIdx]; arr[minIdx] = temp;
    }
}
// Time: O(n²), Space: O(1)

4. Find Missing Number

Approach 1 — Sum Formula (Optimized):

public static int findMissingNumber(int[] arr, int n) {
    int expectedSum = n * (n + 1) / 2;
    int actualSum = 0;
    for (int num : arr) actualSum += num;
    return expectedSum - actualSum;
}
// findMissingNumber({1,2,4,5}, 5) → 3 | Time: O(n), Space: O(1)

Approach 2 — XOR (Bit Manipulation):

public static int findMissingXOR(int[] arr, int n) {
    int xor1 = 0, xor2 = 0;
    for (int i = 1; i <= n; i++) xor1 ^= i;
    for (int num : arr) xor2 ^= num;
    return xor1 ^ xor2;
}
// Same result, O(n) time, O(1) space — no overflow risk for large n

5. Remove Duplicates

Approach 1 — LinkedHashSet (Optimized for unsorted):

public static int[] removeDuplicates(int[] arr) {
    return Arrays.stream(arr).distinct().toArray();
    // Or: LinkedHashSet preserves order
}

Approach 2 — Two Pointer (for sorted arrays):

public static int removeDuplicatesSorted(int[] arr) {
    if (arr.length == 0) return 0;
    int uniqueIndex = 0;
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] != arr[uniqueIndex]) {
            arr[++uniqueIndex] = arr[i];
        }
    }
    return uniqueIndex + 1;  // new length
}
// removeDuplicates({1,1,2,2,3}) → length 3: [1,2,3,...] | Time: O(n)

6. Find Duplicates

Approach 1 — HashSet:

public static Set<Integer> findDuplicates(int[] arr) {
    Set<Integer> seen = new HashSet<>();
    Set<Integer> duplicates = new HashSet<>();
    for (int num : arr) {
        if (!seen.add(num)) duplicates.add(num);
    }
    return duplicates;
}
// findDuplicates({1,2,3,2,4,3,5}) → [2, 3]

Approach 2 — Frequency Map:

public static List<Integer> findDuplicatesFrequency(int[] arr) {
    Map<Integer, Integer> freq = new HashMap<>();
    for (int num : arr) freq.merge(num, 1, Integer::sum);
    return freq.entrySet().stream()
        .filter(e -> e.getValue() > 1)
        .map(Map.Entry::getKey)
        .collect(Collectors.toList());
}

7. Reverse Array

Approach 1 — Two Pointer Swap (Optimized):

public static void reverseArray(int[] arr) {
    int left = 0, right = arr.length - 1;
    while (left < right) {
        int temp = arr[left];
        arr[left] = arr[right];
        arr[right] = temp;
        left++; right--;
    }
}
// reverseArray({1,2,3,4,5}) → [5,4,3,2,1] | Time: O(n), Space: O(1)

Approach 2 — Collections.reverse with Arrays.asList:

public static void reverseArrayList(String[] arr) {
    List<String> list = new ArrayList<>(Arrays.asList(arr));
    Collections.reverse(list);
    // Note: works for object arrays, not primitive int[]
}

When to Use Arrays vs Collections in Frameworks

  • Use Arrays when: TestNG DataProvider returns fixed test data, browser list is known at compile time, storing By[] locators for batch verification, performance matters (large data processing)
  • Use ArrayList when: Collecting WebElements dynamically (findElements), building test result lists at runtime, storing API response items of unknown count, need add/remove during test execution
  • Use HashMap when: Key-value test data (username → password), config properties, API response field extraction

Interview Q&A — Arrays

Q1: What is the default value of array elements?

A: 0 for numeric types, false for boolean, null for object references. int[] arr = new int[5] → [0,0,0,0,0].

Q2: Can you change array size after creation?

A: No. Arrays are fixed-size. Use ArrayList for dynamic sizing. Or Arrays.copyOf() to create a larger copy.

Q3: What does TestNG DataProvider return?

A: Object[][] — a 2D array where each row is a set of test parameters. One test method runs once per row.

Q4: Array vs ArrayList — when to use which in automation?

A: Arrays for fixed DataProvider data and browser lists. ArrayList for dynamic element collections from findElements() and runtime test results.

Q5: How to sort an array of test execution times?

A: Arrays.sort(times) for primitive int[]. For objects: Arrays.sort(results, Comparator.comparing(r -> r.getDuration())).

Q6: What is ArrayIndexOutOfBoundsException?

A: Accessing index < 0 or >= array.length. Common when iterating DataProvider data with wrong index or accessing empty findElements() result.

Q7: Can arrays store different types?

A: Object[] can store any type (with casting). DataProvider uses Object[][] for mixed types (String, int, boolean). Primitive arrays are type-specific.

Q8: How to convert array to List?

A: Arrays.asList(array) — fixed-size list. For mutable: new ArrayList<>(Arrays.asList(array)). For primitives: IntStream.of(arr).boxed().collect(Collectors.toList()).

Q9: How to find array length?

A: arr.length (property, not method — no parentheses). For 2D: arr.length gives rows, arr[0].length gives columns.

Q10: What is the difference between length and length()?

A: arr.length — array property (no parentheses). str.length() — String method. list.size() — Collection method. Common interview trick question.
