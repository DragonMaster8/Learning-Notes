Q1. Reverse a String

FieldDetails
CompanyInfosys, TCS, Wipro
QuestionReverse a given string and return the result.
ApproachTwo pointers on char array swapping from both ends; or StringBuilder.reverse().
Time ComplexityO(n)
Space ComplexityO(n) for new char array
Interview TipMention String immutability — cannot reverse in-place on String object itself.

Step-by-Step Explanation

  1. Convert String to char[] for mutability.
  2. Set left=0, right=length-1; swap and move inward.
  3. Return new String from char array.

Complete Solution Code

public class ReverseString {
    public static String reverse(String input) {
        if (input == null) return null;
        char[] chars = input.toCharArray();
        int left = 0, right = chars.length - 1;
        while (left < right) {
            char temp = chars[left];
            chars[left++] = chars[right];
            chars[right--] = temp;
        }
        return new String(chars);
    }
    public static void main(String[] args) {
        System.out.println(reverse("automation")); // noitamotua
    }
}

Alternative Approach

new StringBuilder(input).reverse().toString() — one line but allocates StringBuilder object.

Q2. Check Palindrome

FieldDetails
CompanyInfosys, TCS, Wipro
QuestionDetermine if a string reads the same forward and backward.
ApproachTwo-pointer comparison; optionally skip non-alphanumeric and ignore case.
Time ComplexityO(n)
Space ComplexityO(1)
Interview TipClarify whether spaces and punctuation matter — common follow-up.

Step-by-Step Explanation

  1. Use two pointers from both ends.
  2. Skip non-alphanumeric characters if required.
  3. Compare lowercase chars; return false on mismatch.

Complete Solution Code

public class PalindromeCheck {
    public static boolean isPalindrome(String s) {
        if (s == null) return false;
        int l = 0, r = s.length() - 1;
        while (l < r) {
            while (l < r && !Character.isLetterOrDigit(s.charAt(l))) l++;
            while (l < r && !Character.isLetterOrDigit(s.charAt(r))) r--;
            if (Character.toLowerCase(s.charAt(l++)) != Character.toLowerCase(s.charAt(r--)))
                return false;
        }
        return true;
    }
    public static void main(String[] args) {
        System.out.println(isPalindrome("A man a plan a canal Panama"));
    }
}

public class Factorial {
    public static long factorial(int n) {
        if (n < 0) throw new IllegalArgumentException("n must be >= 0");
        long result = 1;

Q4. Fibonacci Series

FieldDetails
CompanyInfosys, TCS, Wipro
QuestionPrint first n Fibonacci numbers or return nth value.
ApproachIterative with two variables — optimal for interviews.
Time ComplexityO(n)
Space ComplexityO(1)
Interview TipAvoid naive recursion O(2^n); mention memoization if asked.

Step-by-Step Explanation

  1. Initialize a=0, b=1.
  2. Loop n times: print a, then shift a,b to next pair.
  3. For nth value, loop until index n.

Complete Solution Code

public class Fibonacci {
    public static void printSeries(int n) {
        int a = 0, b = 1;
        for (int i = 0; i < n; i++) {
            System.out.print(a + " ");
            int next = a + b; a = b; b = next;
        }
    }
    public static int nth(int n) {
        if (n <= 1) return n;
        int a = 0, b = 1;
        for (int i = 2; i <= n; i++) {
            int next = a + b; a = b; b = next;
        }
        return b;
    }
    public static void main(String[] args) {
        printSeries(10);
        System.out.println("\n10th: " + nth(10));
    }
}

Q9. Sort Array (Bubble Sort)

FieldDetails
CompanyInfosys, TCS, Wipro
QuestionSort an integer array in ascending order.
ApproachBubble sort — swap adjacent elements; early exit if no swaps.
Time ComplexityO(n²)
Space ComplexityO(1)
Interview TipMention Arrays.sort() uses TimSort O(n log n) in production code.

Step-by-Step Explanation

  1. Outer loop for number of passes.
  2. Inner loop compares arr[j] and arr[j+1]; swap if out of order.
  3. Break early if no swaps in a pass.

Complete Solution Code

import java.util.Arrays;
public class BubbleSort {
    public static void sort(int[] arr) {
        for (int i = 0; i < arr.length - 1; i++) {
            boolean swapped = false;
            for (int j = 0; j < arr.length - 1 - i; j++) {
                if (arr[j] > arr[j + 1]) {
                    int t = arr[j]; arr[j] = arr[j + 1]; arr[j + 1] = t;
                    swapped = true;
                }
            }
            if (!swapped) break;
        }
    }
    public static void main(String[] args) {
        int[] a = {5, 1, 4, 2};
        sort(a);
        System.out.println(Arrays.toString(a));
    }
}

Q11. Find Second Largest in Array

FieldDetails
CompanyCognizant, Capgemini, Accenture
QuestionFind the second largest distinct element in an array.
ApproachTrack largest and secondLargest in one pass.
Time ComplexityO(n)
Space ComplexityO(1)
Interview TipHandle duplicates and arrays with fewer than 2 distinct elements.

Step-by-Step Explanation

  1. Initialize first and second to Integer.MIN_VALUE.
  2. Update both when finding a new maximum.
  3. Return second; throw if not found.

Complete Solution Code

public class SecondLargest {
    public static int findSecond(int[] arr) {
        int first = Integer.MIN_VALUE, second = Integer.MIN_VALUE;
        for (int n : arr) {
            if (n > first) { second = first; first = n; }
            else if (n > second && n != first) second = n;
        }
        if (second == Integer.MIN_VALUE) throw new IllegalStateException("No second largest");
        return second;
    }
    public static void main(String[] args) {
        System.out.println(findSecond(new int[]{12, 35, 1, 10, 35}));
    }
}

import java.util.*;
public class TwoSum {
    public static int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();
        for (int i = 0; i < nums.length; i++) {
            int need = target - nums[i];

Time ComplexityO(m + n)
Space ComplexityO(m + n)

import java.util.Arrays;
public class MergeSortedArrays {
    public static int[] merge(int[] a, int[] b) {
        int[] result = new int[a.length + b.length];
        int i = 0, j = 0, k = 0;
        while (i < a.length && j < b.length)
            result[k++] = (a[i] <= b[j]) ? a[i++] : b[j++];
        while (i < a.length) result[k++] = a[i++];
        while (j < b.length) result[k++] = b[j++];
        return result;
    }
    public static void main(String[] args) {
        System.out.println(Arrays.toString(merge(new int[]{1,3,5}, new int[]{2,4,6})));

Q21. Longest Substring Without Repeating

FieldDetails
CompanyEPAM, Globant, LTIMindtree
QuestionFind length of longest substring with all unique characters.
ApproachSliding window with HashSet; shrink on duplicate.
Time ComplexityO(n)
Space ComplexityO(min(n, m)) charset size
Interview TipSliding window pattern applies to rate-limit testing windows.

Complete Solution Code

import java.util.*;
public class LongestSubstring {
    public static int lengthOfLongest(String s) {
        Set<Character> set = new HashSet<>();
        int max = 0, left = 0;
        for (int right = 0; right < s.length(); right++) {
            while (set.contains(s.charAt(right))) set.remove(s.charAt(left++));
            set.add(s.charAt(right));
            max = Math.max(max, right - left + 1);
        }
        return max;
    }
    public static void main(String[] args) {
        System.out.println(lengthOfLongest("abcabcbb"));
    }
}

Q22. Valid Parentheses (Stack)

FieldDetails
CompanyEPAM, Globant, LTIMindtree
QuestionValidate balanced brackets: (), {}, [].
ApproachStack push opening; pop and match on closing.
Time ComplexityO(n)
Space ComplexityO(n)
Interview TipStack models nested TestNG group/suite structure.

Complete Solution Code

import java.util.*;
public class ValidParentheses {
    public static boolean isValid(String s) {
        Map<Character, Character> map = Map.of(')', '(', '}', '{', ']', '[');
        Deque<Character> stack = new ArrayDeque<>();
        for (char c : s.toCharArray()) {
            if (map.containsValue(c)) stack.push(c);
            else if (map.containsKey(c)) {
                if (stack.isEmpty() || stack.pop() != map.get(c)) return false;
            }
        }
        return stack.isEmpty();
    }
    public static void main(String[] args) {
        System.out.println(isValid("{[()]}"));
    }
}

import java.util.Arrays;
public class RotateArray {
    private static void reverse(int[] a, int l, int r) {
        while (l < r) { int t = a[l]; a[l++] = a[r]; a[r--] = t; }

import java.util.*;

        for (String w : words) {

Q32. Three Sum Problem

FieldDetails

QuestionFind all unique triplets summing to zero.

Space ComplexityO(1) excluding output

import java.util.*;
public class ThreeSum {
    public static List<List<Integer>> threeSum(int[] nums) {
        Arrays.sort(nums);
        List<List<Integer>> res = new ArrayList<>();
        for (int i = 0; i < nums.length - 2; i++) {
            if (i > 0 && nums[i] == nums[i-1]) continue;
            int lo = i + 1, hi = nums.length - 1;
            while (lo < hi) {
                int sum = nums[i] + nums[lo] + nums[hi];
                if (sum == 0) {
                    res.add(Arrays.asList(nums[i], nums[lo], nums[hi]));
                    while (lo < hi && nums[lo] == nums[lo+1]) lo++;
                    while (lo < hi && nums[hi] == nums[hi-1]) hi--;
                    lo++; hi--;
                } else if (sum < 0) lo++; else hi--;
            }
        }
        return res;
    }

    public static int[] maxSlidingWindow(int[] nums, int k) {
        Deque<Integer> dq = new ArrayDeque<>();
        int[] result = new int[nums.length - k + 1];
        for (int i = 0; i < nums.length; i++) {
            while (!dq.isEmpty() && dq.peekFirst() <= i - k) dq.pollFirst();

        }

Q34. LRU Cache Design

FieldDetails
CompanyGoogle, Amazon
QuestionDesign LRU cache with O(1) get and put.
ApproachHashMap + doubly linked list for order.
Time ComplexityO(1) per operation
Space ComplexityO(capacity)
Interview TipSame pattern as caching Page Object elements or API responses.

Complete Solution Code

import java.util.*;
public class LRUCache {
    class Node { int key, val; Node prev, next; Node(int k, int v){key=k;val=v;} }
    private final int cap;
    private final Map<Integer, Node> map = new HashMap<>();
    private final Node head = new Node(0,0), tail = new Node(0,0);
    public LRUCache(int capacity) { cap = capacity; head.next = tail; tail.prev = head; }
    public int get(int key) {
        if (!map.containsKey(key)) return -1;
        Node n = map.get(key); moveToHead(n); return n.val;
    }
    public void put(int key, int value) {
        if (map.containsKey(key)) { map.get(key).val = value; moveToHead(map.get(key)); }
        else {
            Node n = new Node(key, value); map.put(key, n); addToHead(n);
            if (map.size() > cap) { Node rem = tail.prev; remove(rem); map.remove(rem.key); }
        }
    }
    private void moveToHead(Node n){ remove(n); addToHead(n); }
    private void addToHead(Node n){ n.prev=head; n.next=head.next; head.next.prev=n; head.next=n; }
    private void remove(Node n){ n.prev.next=n.next; n.next.prev=n.prev; }
}

Q38. Word Ladder Problem

FieldDetails
CompanyGoogle, Amazon
QuestionShortest transformation from beginWord to endWord.
ApproachBFS — try changing each char to a-z.
Time ComplexityO(M² × N)
Space ComplexityO(M × N)
Interview TipExplain BFS shortest path with queue + visited set.

Complete Solution Code

import java.util.*;
public class WordLadder {
    public static int ladderLength(String begin, String end, List<String> wordList) {
        Set<String> dict = new HashSet<>(wordList);
        if (!dict.contains(end)) return 0;
        Queue<String> q = new LinkedList<>();
        q.offer(begin);
        int level = 1;
        while (!q.isEmpty()) {
            int size = q.size();
            for (int i = 0; i < size; i++) {
                char[] word = q.poll().toCharArray();
                for (int j = 0; j < word.length; j++) {
                    char orig = word[j];
                    for (char c = 'a'; c <= 'z'; c++) {
                        word[j] = c;
                        String next = new String(word);
                        if (next.equals(end)) return level + 1;
                        if (dict.contains(next)) { q.offer(next); dict.remove(next); }
                    }
                    word[j] = orig;
                }
            }
            level++;
        }
        return 0;
    }
}

Top 25 Most Asked Java Questions for SDET

#Question2-Line Answer
1What is JDK vs JRE vs JVM?JDK compiles and develops; JRE runs apps; JVM executes bytecode. Automation needs JDK on CI for Maven compilation.
2Why is String immutable?Security (credentials), thread safety (String pool), and hash stability. Use StringBuilder for loop concatenation in report builders.
3== vs .equals()?== compares references; equals() compares content. Always use equals() for String comparison in assertions.
4What is method overloading?Same method name, different parameters — compile time. Example: click(By) and click(WebElement) in BasePage.
5What is method overriding?Subclass redefines parent method — runtime polymorphism. Example: @BeforeMethod setup() in child BaseTest.
6Abstract class vs Interface?Abstract class: partial implementation + state. Interface: contract only (Java 8+ default methods). DriverFactory often uses interface.
7What is encapsulation?Hide fields, expose via getters/setters. Page Objects hide WebElement locators from test classes.
8Checked vs Unchecked exception?Checked: must handle (IOException). Unchecked: RuntimeException (NoSuchElementException). Selenium throws mostly unchecked.
9What is final keyword?final class = no inheritance; final method = no override; final variable = constant. Config URLs often final static.
10What is static keyword?Belongs to class, not instance. ConfigReader.get() is static — shared across all tests in JVM.
11How does HashMap work internally?Array of buckets; hashCode → bucket index; collision via linked list/tree (Java 8). O(1) average get/put.
12ArrayList vs LinkedList?ArrayList: fast random access O(1). LinkedList: fast insert/delete O(1). Use ArrayList for test name lists.
13What is ThreadLocal?Thread-bound variable storage. Each TestNG thread gets its own WebDriver instance via ThreadLocal.
14What is synchronized?Lock for thread-safe access. Rare in tests; prefer ThreadLocal over synchronized WebDriver access.
15What are Java 8 lambdas?Anonymous functions enabling functional-style code. Filter failed tests: results.stream().filter(r -> r.isFailed()).
16What is Optional?Container avoiding NullPointerException. findFirst() returns Optional — use orElse/orElseThrow safely.
17Comparable vs Comparator?Comparable: natural order (implements compareTo). Comparator: custom order. Sort tests by duration with Comparator.
18What is garbage collection?JVM reclaims unreferenced objects. driver.quit() releases browser; null references help GC reclaim memory.
19What is autoboxing?Primitive ↔ wrapper automatic conversion. int ↔ Integer in collections and stream operations.
20What is serialization?Object → byte stream for storage/transmission. Save Cucumber test context or session cookies to file.
21What is reflection?Inspect/modify classes at runtime. TestNG uses reflection to invoke @Test methods; read custom annotations.
22What is Singleton pattern?One instance per JVM. ConfigReader and ReportManager — but NOT WebDriver in parallel (use ThreadLocal).
23What is Factory pattern?Centralized object creation. DriverFactory.createDriver(browser) returns correct WebDriver implementation.
24What is Builder pattern?Step-by-step object construction. TestConfig.Builder().browser("chrome").url(url).build().
25What is Page Object Model?UI locators + actions in page classes; tests call business methods. Reduces duplication and maintenance cost.

50 Quick Revision Points

1–10: OOP Essentials

  1. Four pillars: Encapsulation, Inheritance, Polymorphism, Abstraction.
  2. Page Object Model is encapsulation applied to UI automation.
  3. WebDriver is an interface — ChromeDriver/FirefoxDriver are implementations.
  4. Overloading: same name, different parameters — compile time binding.
  5. Overriding: subclass method replaces parent — runtime binding with @Override.
  6. Abstract class can have constructors; interface cannot (until Java changes).
  7. Java supports single inheritance for classes; multiple interfaces allowed.
  8. super() calls parent constructor; super.method() calls parent method.
  9. this() calls another constructor in same class.
  10. Composition (has-a) preferred over deep inheritance for test utilities.

11–20: String Essentials

  1. String is immutable — every modification creates new object.
  2. String pool interns literal strings for memory efficiency.
  3. StringBuilder for single-thread string building in loops.
  4. String.join(",", list) replaces manual comma concatenation.
  5. String.format("//a[text()='%s']", name) for dynamic XPath.
  6. split("\\s+") handles multiple spaces in log parsing.
  7. equalsIgnoreCase for browser name from config comparison.
  8. trim() before comparing Excel cell values.
  9. isBlank() (Java 11) checks empty or whitespace-only strings.
  10. Never use == for String content comparison in assertions.

21–30: Collections Essentials

  1. ArrayList: default choice for ordered test case lists.
  2. HashMap: default for key-value test data and config cache.
  3. HashSet: deduplicate test IDs or tags.
  4. LinkedHashMap preserves Excel column order in row maps.
  5. ConcurrentHashMap for thread-safe shared state in parallel runs.
  6. Iterator.remove() safe during iteration; avoid ConcurrentModificationException.
  7. Collections.sort() vs list.sort(Comparator) — prefer latter in Java 8+.
  8. Map.getOrDefault(key, default) avoids null checks on config keys.
  9. Map.merge(key, 1, Integer::sum) for frequency counting.
  10. Arrays.asList() returns fixed-size list — use new ArrayList<>() if mutating.

31–40: Java 8 Essentials

  1. Lambda: (param) -> expression — replaces anonymous inner classes.
  2. Method reference: TestResult::getName shorthand for t -> t.getName().
  3. Stream pipeline: source → intermediate → terminal operation.
  4. filter + map + collect is the most common stream pattern in reports.
  5. Predicate<T> for filtering; Function<T,R> for mapping.
  6. Optional.ofNullable avoids NullPointerException on findFirst results.
  7. Collectors.groupingBy groups test results by status/pass/fail.
  8. Collectors.toMap builds lookup from test name to TestResult.
  9. forEach is terminal — use for side effects like logging.
  10. Parallel streams rarely help in Selenium tests (I/O bound, not CPU bound).

41–50: Framework / Design Essentials

  1. ThreadLocal WebDriver: set → get → quit → remove per test thread.
  2. Singleton ConfigReader: load properties once in static block.
  3. Factory pattern: DriverFactory.createDriver(browser) centralizes creation.
  4. Builder pattern: fluent TestConfig construction with validation.
  5. Page Object: @FindBy locators + public action methods + no assertions in page.
  6. BaseTest: @BeforeMethod driver setup, @AfterMethod teardown + screenshot on fail.
  7. Explicit waits (WebDriverWait) — never Thread.sleep in framework code.
  8. DataProvider returns Object[][] from Excel/JSON reader utility.
  9. IRetryAnalyzer for flaky test retry with max attempt limit.
  10. Layered framework: Tests → Page Objects → Utilities → Config/Data.

Common Interview Mistakes for 4+ Years

  1. Saying "I know Java" but can't write code — always practice on whiteboard/paper.
  2. Not explaining framework architecture — draw layers: config, driver, pages, tests, reports.
  3. Confusing overloading/overriding — overload = same class; override = inheritance.
  4. Not knowing HashMap internal working — hashCode, bucket, collision chain/tree.
  5. Can't explain ThreadLocal — per-thread WebDriver isolation for parallel TestNG.
  6. Not knowing design patterns — Singleton, Factory, Builder, Page Object minimum.
  7. Poor coding — no edge cases (null, empty), no error handling.
  8. Not discussing time complexity — always state Big-O for coding solutions.
  9. Not relating Java to automation — connect every concept to framework usage.
  10. Using deprecated practices — findElementBy* instead of By locators; Thread.sleep.
  11. Not knowing Java 8 features — streams, lambdas, Optional expected at 4+ years.
  12. Poor problem-solving approach — clarify input/output, examples, edge cases before coding.
