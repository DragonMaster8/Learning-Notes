Why Collections in Automation?

  • Dynamic test data — CSV rows, Excel sheets, JSON arrays loaded into List or Map
  • Config storage — Map<String, String> for properties, environment variables, API headers

List Interface

Ordered, allows duplicates, index-based access. The two workhorses in automation are ArrayList and LinkedList.

ArrayList

Internal structure: Backed by a dynamic array. Default initial capacity is 10. When full, it grows by approximately 50% (creates a new array and copies elements).

Automation usage:

  • Store WebElement lists from findElements()
  • Hold test data rows (usernames, product IDs)
  • Collect assertion failure messages for soft asserts

    card.findElement(By.cssSelector(".add-btn")).click();
}

// Test data list
List<String> usernames = Arrays.asList("admin", "user1", "guest");

Internal structure: Doubly linked list — each node holds data + references to previous and next nodes.

Q1: Remove Duplicates from ArrayList Using Set

Problem: Given a list of test URLs with duplicates, return a list with unique entries preserving insertion order.

Approach: Pass the ArrayList to a LinkedHashSet constructor (removes dupes, keeps order), then create a new ArrayList from the set.

import java.util.*;

public class RemoveDuplicates {
    public static void main(String[] args) {
        List<String> urls = new ArrayList<>(Arrays.asList(
            "/login", "/dashboard", "/login", "/settings", "/dashboard"
        ));

        // LinkedHashSet preserves insertion order
        Set<String> uniqueSet = new LinkedHashSet<>(urls);
        List<String> uniqueList = new ArrayList<>(uniqueSet);

        System.out.println("Original: " + urls);
        System.out.println("Unique:   " + uniqueList);
    }
}

Output:

Original: [/login, /dashboard, /login, /settings, /dashboard]
Unique:   [/login, /dashboard, /settings]

Complexity: Time O(n), Space O(n)

Q2: Find Duplicate Elements

Problem: Find all duplicate test case IDs in a list.

Approach: HashSet.add() returns false when the element already exists — that's a duplicate.

import java.util.*;

public class FindDuplicates {
    public static void main(String[] args) {
        List<String> testIds = Arrays.asList("TC001", "TC002", "TC001", "TC003", "TC002", "TC004");

        Set<String> seen = new HashSet<>();
        Set<String> duplicates = new HashSet<>();

        for (String id : testIds) {
            if (!seen.add(id)) {  // add returns false = duplicate
                duplicates.add(id);
            }
        }

        System.out.println("Duplicate test IDs: " + duplicates);
    }
}

Output:

Duplicate test IDs: [TC001, TC002]

Complexity: Time O(n), Space O(n)

Q4: Sort List of Employees (Comparable vs Comparator)

Problem: Sort automation team members by salary, then by name.

import java.util.*;

class Employee implements Comparable<Employee> {
    String name;
    int salary;

    Employee(String name, int salary) {
        this.name = name;
        this.salary = salary;
    }

    // Natural ordering: by salary ascending
    @Override
    public int compareTo(Employee other) {
        return Integer.compare(this.salary, other.salary);
    }

    @Override
    public String toString() {
        return name + " ($" + salary + ")";
    }
}

public class SortEmployees {
    public static void main(String[] args) {
        List<Employee> team = new ArrayList<>(Arrays.asList(
            new Employee("Alice", 90000),
            new Employee("Bob", 75000),
            new Employee("Charlie", 90000)
        ));

        // Approach 1: Comparable (natural order — by salary)
        Collections.sort(team);
        System.out.println("By salary (Comparable): " + team);

        // Approach 2: Comparator (custom — by name)
        team.sort(Comparator.comparing(e -> e.name));
        System.out.println("By name (Comparator):   " + team);

        // Approach 3: Comparator chaining — salary desc, then name asc
        team.sort(Comparator.comparingInt((Employee e) -> e.salary).reversed()
                          .thenComparing(e -> e.name));
        System.out.println("Salary desc, name asc:  " + team);
    }
}

Output:

By salary (Comparable): [Bob ($75000), Alice ($90000), Charlie ($90000)]
By name (Comparator):   [Alice ($90000), Bob ($75000), Charlie ($90000)]
Salary desc, name asc:  [Alice ($90000), Charlie ($90000), Bob ($75000)]

Complexity: Time O(n log n), Space O(1)

Q5: Sort HashMap by Value

Problem: Sort test failure counts by frequency (value) descending.

Approach: Stream API on entrySet(), sort by value, collect to LinkedHashMap.

import java.util.*;
import java.util.stream.*;

public class SortMapByValue {
    public static void main(String[] args) {
        Map<String, Integer> failures = new HashMap<>();
        failures.put("testLogin", 15);
        failures.put("testCheckout", 8);
        failures.put("testSearch", 22);
        failures.put("testProfile", 3);

        Map<String, Integer> sorted = failures.entrySet().stream()
            .sorted(Map.Entry.<String, Integer>comparingByValue().reversed())
            .collect(Collectors.toMap(
                Map.Entry::getKey,
                Map.Entry::getValue,
                (a, b) -> a,
                LinkedHashMap::new
            ));

        sorted.forEach((test, count) ->
            System.out.println(test + " → " + count + " failures")
        );
    }
}

Output:

testSearch → 22 failures
testLogin → 15 failures
testCheckout → 8 failures
testProfile → 3 failures

Complexity: Time O(n log n), Space O(n)

Q6: Count Word Frequency in Paragraph

Problem: Count word frequency in an error log paragraph to find most common errors.

import java.util.*;

public class WordFrequency {
    public static void main(String[] args) {
        String log = "timeout timeout element not found stale element timeout connection refused";

        Map<String, Integer> wordCount = new HashMap<>();
        for (String word : log.split("\\s+")) {
            word = word.toLowerCase();
            wordCount.put(word, wordCount.getOrDefault(word, 0) + 1);
        }

        // Find most frequent word
        String maxWord = Collections.max(wordCount.entrySet(),
            Map.Entry.comparingByValue()).getKey();

        System.out.println("Word frequencies: " + wordCount);
        System.out.println("Most frequent:    " + maxWord);
    }
}

Output:

Word frequencies: {timeout=3, element=2, not=1, found=1, stale=1, connection=1, refused=1}
Most frequent:    timeout

Complexity: Time O(n), Space O(k) where k = unique words

Interview Q&A — Section 6 (15+ Questions)

Q: What is the difference between Collection and Collections?

A: Collection is the root interface of the framework hierarchy. Collections is a utility class with static methods like sort(), reverse(), synchronizedList(). In automation, you use Collections.sort() to sort test data lists.

Q: Why is ArrayList preferred over Vector?

A: Vector is synchronized (thread-safe but slow). ArrayList is faster for single-threaded test execution. For parallel tests, use thread-safe alternatives like CopyOnWriteArrayList, not Vector.

Q: Can we store null in ArrayList?

A: Yes. ArrayList allows multiple nulls. Useful when parsing optional fields from test data where some columns may be empty.

Q: How does HashSet maintain uniqueness?

A: It uses hashCode() to find the bucket and equals() to compare within the bucket. If equals() returns true, the element is not added. Override both in custom test data objects.

Q: What happens if two keys in HashMap have the same hashCode?

A: Collision occurs. Java stores them in a linked list within the same bucket. If the bucket exceeds 8 entries (Java 8+), it converts to a Red-Black tree for O(log n) performance.

Q: Why doesn't HashMap implement Collection interface?

A: Map stores key-value pairs, not individual elements. It has its own hierarchy with Map.Entry for iterating entries.

Q: When would you use LinkedHashMap over HashMap in a framework?

A: When config property order matters — e.g., test execution order defined in a properties file, or maintaining insertion order of API request parameters for debugging.

Q: Is HashMap thread-safe?

A: No. In parallel TestNG/Cucumber execution, use ConcurrentHashMap for shared config or report data.

Q: What is fail-fast vs fail-safe iterator?

A: Fail-fast (ArrayList, HashMap) throws ConcurrentModificationException on structural modification during iteration. Fail-safe (ConcurrentHashMap) works on a clone — no exception but may not reflect latest changes.

Q: How do you convert ArrayList to Array and vice versa?

A: list.toArray(new String[0]) and new ArrayList<>(Arrays.asList(array)) or Arrays.asList() (fixed-size).

Q: What is the initial capacity of ArrayList and why does it matter?

A: Default is 10. If you know you'll store 500 test data rows, set initial capacity to avoid repeated resizing: new ArrayList<>(500).

Q: Comparable vs Comparator — when to use which?

A: Comparable = natural ordering defined inside the class (one sort logic). Comparator = external, multiple sort strategies without modifying the class. In frameworks, use Comparator for flexible test data sorting.

Q: How do you iterate a HashMap?

A: Four ways: (1) entrySet() — best, (2) keySet() then get value, (3) values(), (4) Java 8 forEach. Prefer entrySet() for config iteration.

Q: What is ConcurrentModificationException and how to avoid it?

A: Thrown when a collection is structurally modified during iteration (except via Iterator.remove()). In automation, common when filtering WebElements during for-each. Fix: use Iterator, stream filter, or removeAll with a separate list.

Q: Can TreeMap have null keys?

A: No. TreeMap uses comparison — null key causes NullPointerException. Use HashMap if null keys are needed.

Q: How would you find the intersection of two lists of test data?

A: list1.retainAll(list2) modifies list1 to keep only common elements. Or use streams: list1.stream().filter(list2::contains).collect(Collectors.toList()).

Q: What collection would you use to store WebElements and why?

A: ArrayList<WebElement> — because findElements() returns a list, you need index access for specific rows, and you iterate sequentially. No need for Set (duplicates aren't an issue) or Map (no key-value relationship).

Exception Handling Best Practices for Frameworks

  1. Catch specific, not generic — catch NoSuchElementException, not Exception
  2. Always log before re-throwing — include context (page name, locator, test name)
  3. Take screenshot on failure — in catch block or TestNG listener
  4. Use custom exceptions — wrap Selenium exceptions with business meaning
  5. Never swallow exceptions — empty catch blocks hide failures
  6. Clean up in finally or @AfterMethod — quit driver, close connections
  7. try-with-resources — for file streams, readers, WebDriver (Selenium 4+)
  8. Retry for transient failures — stale element, intermittent timeout
  9. Fail fast for config errors — throw ConfigException at startup, not mid-test
  10. Use TestNG listeners — centralized exception handling, reporting, screenshots

Interview Q&A — Section 7 (15+ Questions)

Q: What is the difference between throw and throws?

A: throw explicitly creates and throws an exception instance inside a method. throws declares in the method signature that checked exceptions may propagate to the caller.

Q: Can we have try without catch?

A: Yes — try with finally only, or try-with-resources (Java 7+). Both are valid.

Q: Does finally always execute?

A: Almost always. Exceptions: System.exit(), JVM crash, infinite loop in try, daemon thread killed.

Q: What happens if catch and finally both throw exceptions?

A: The exception from finally suppresses the one from catch. The catch exception is added as suppressed (Java 7+).

Q: Why are Selenium exceptions unchecked?

A: They extend RuntimeException. Selenium doesn't force you to handle them at compile time — you handle them at the framework design level with waits and retry utilities.

Q: How do you handle StaleElementReferenceException in a framework?

A: Implement a retry wrapper in BasePage that re-finds the element and retries the action up to N times before failing.

Q: What is exception chaining?

A: Wrapping one exception inside another: throw new FrameworkException("Login failed", originalException). Preserves the root cause stack trace.

Q: Can we catch Error?

A: Technically yes, but you shouldn't. Errors (OutOfMemoryError, StackOverflowError) indicate serious JVM problems that tests cannot recover from.

Q: What is multi-catch (Java 7+)?

A: catch (NoSuchElementException | TimeoutException e) — single catch block for multiple exception types.

Q: How do you handle exceptions in TestNG?

A: (1) try-catch in test/page methods, (2) @Test(expectedExceptions=...) for negative tests, (3) ITestListener.onTestFailure() for screenshots/reports, (4) @Test(retryAnalyzer=...) for flaky test retry.

Q: What is the difference between Error and Exception?

A: Error = serious JVM problems (not recoverable). Exception = conditions apps can handle (file not found, element not found).

Q: When would you use checked exceptions in automation?

A: Reading config files (IOException), database connections (SQLException), parsing XML test data. Wrap in custom ConfigException or TestDataException for cleaner framework API.

Q: How do you log exceptions properly?

A: logger.error("Failed to click login button on page: {}", pageName, exception) — include context message AND pass the exception object for full stack trace.

Q: What is try-with-resources?

A: Java 7 feature that auto-closes AutoCloseable resources. Compiler generates finally block with close(). Use for file readers, DB connections, WebDriver.

Q: How do you design exception handling in a Page Object Model?

A: BasePage provides safe action methods (safeClick, safeSendKeys) with built-in waits and retry. Page-specific methods throw custom exceptions (PageNotFoundException). Tests never catch Selenium exceptions directly.

Q: Can finally block change the return value?

A: Yes — if finally has a return statement, it overrides try/catch return value. Avoid return in finally — it's a bad practice.

  • Internal helper methods (e.g., private void waitForAjax())

Visible within the same package only. Used for package-level utilities and internal test helpers.

}

Accessible to subclasses — the backbone of BasePage/BaseTest inheritance.

public abstract class BasePage {

Interview Q&A — Section 8 (10+ Questions)

Q: What is the most restrictive access modifier?

A: private — accessible only within the same class.

Q: Can a top-level class be private?

A: No. Top-level classes can only be public or package-private (default). private applies to inner/nested classes.

Q: Why are locators private in Page Objects?

A: Encapsulation — if a locator changes, you update one private field. Tests use public action methods and never reference locators directly.

Q: protected vs default — what's the difference?

A: Both are accessible in the same package. protected additionally allows access from subclasses in other packages. Critical for BasePage in com.framework.base extended by pages in com.framework.pages.

Q: Can constructors be private?

A: Yes — used in Singleton pattern (e.g., DriverManager) and utility classes to prevent instantiation.

Q: What access modifier for TestNG @Test methods?

A: public (convention) or package-private. TestNG can invoke non-private methods. Never private.

Q: How do access modifiers relate to encapsulation?

A: Encapsulation = hide internal state, expose public API. Access modifiers enforce it: private fields, public getters/setters, protected for inheritance hierarchy.

Q: Can an interface method be protected?

A: Pre-Java 9: all interface methods are implicitly public. Java 9+ allows private methods in interfaces.

Q: Why use protected for BaseTest @BeforeMethod?

A: So child test classes in the same or different package can override setup/teardown while keeping it hidden from unrelated classes.

Q: What happens if DriverManager.driver is public?

A: Any class can directly modify the driver reference — breaks encapsulation, causes thread-safety issues in parallel execution, makes debugging harder.

Q: Can we reduce visibility when overriding a method?

A: No. Overridden method cannot have a more restrictive modifier. If parent method is protected, override must be protected or public.

Section 9: Static Keyword

The static keyword belongs to the class, not to any instance. In automation frameworks, static is used for constants, utility methods, and config loading — but misused static WebDriver is the #1 cause of parallel execution failures.

Static Variable

Shared across ALL instances of the class — one copy in memory, class-level scope.

public class FrameworkConfig {
    public static final String BASE_URL = "https://staging.example.com";
    public static final int DEFAULT_TIMEOUT = 10;
    public static final String BROWSER = "chrome";

    // Counter shared across all test instances
    public static int testsExecuted = 0;
}
⚠️ DANGER: Static WebDriver 
public class DriverFactory {
    public static WebDriver driver;  // ❌ NEVER DO THIS
}

In parallel TestNG execution, ALL threads share the same static driver. Thread 1 opens Chrome, Thread 2 overwrites it with Firefox, Thread 1's test fails with wrong browser or stale session.

  PARALLEL EXECUTION WITH static WebDriver (BROKEN):

  Thread-1 (Test A)          Thread-2 (Test B)
       |                          |
       v                          v
  driver = Chrome            driver = Firefox  ← overwrites!
       |                          |
  driver.get("/login")       driver.get("/cart")
       |                          |
  ❌ FAILS — driver now       ✅ Works on Firefox
     points to Firefox        but broke Test A

  SHARED static driver ──→ ONE WebDriver for ALL threads

Solution: Use ThreadLocal<WebDriver> (covered in Section 14) — each thread gets its own driver instance.

Static Method

Called without creating an object. Cannot access instance variables or use this/super.

public class ConfigReader {
    private static Properties prop = new Properties();

    public static String getProperty(String key) {
        return prop.getProperty(key);
        // Cannot access instance fields here
    }

    public static WebDriver createDriver(String browser) {
        switch (browser.toLowerCase()) {
            case "chrome":  return new ChromeDriver();
            case "firefox": return new FirefoxDriver();
            default:        throw new IllegalArgumentException("Unknown browser: " + browser);
        }
    }
}

// Usage — no object needed
String url = ConfigReader.getProperty("base.url");
WebDriver driver = ConfigReader.createDriver("chrome");

Automation usage:

  • Utility methods: ExcelUtils.readSheet(), RandomDataGenerator.generateEmail()
  • Factory methods: DriverFactory.createDriver()
  • Config readers: ConfigReader.getProperty()

Static Block

Executes once when the class is first loaded by JVM — before any static method or object creation.

public class ConfigReader {
    private static Properties prop;

    static {
        prop = new Properties();
        try (FileInputStream fis = new FileInputStream("src/test/resources/config.properties")) {
            prop.load(fis);
            System.out.println("Config loaded successfully");
        } catch (IOException e) {
            throw new ConfigException("Failed to load config.properties", e);
        }
    }

    public static String getProperty(String key) {
        return prop.getProperty(key);
    }
}

Automation usage: Load config once at class load time — avoids re-reading the file for every test method.

import static org.testng.Assert.*;

assertTrue(condition);

Interview Q&A — Section 9 (12+ Questions)

Q: Can we override static methods?

A: No — static methods are hidden, not overridden. Method resolution is compile-time (based on reference type), not runtime polymorphism.

Q: Why is static WebDriver bad for parallel execution?

A: Static = one shared instance across all threads. Parallel tests overwrite each other's driver, causing random failures. Use ThreadLocal instead.

Q: Can a class be static?

A: Top-level classes cannot be static. Only nested classes can be declared static — they don't need an outer class instance.

Q: When does static block execute?

A: Once, when the class is first loaded by ClassLoader — before main(), before any static method call, before first object creation.

Q: Can static methods be synchronized?

A: Yes — locks on the Class object. Used in Singleton double-checked locking. Rare in automation.

Q: What is static variable default value?

A: Default initialized by JVM: 0 for numbers, false for boolean, null for objects. Instance variables get defaults too, but static has one shared copy.

Q: Why do utility classes have private constructors?

A: To prevent instantiation — all methods are static, so creating an object is meaningless. Example: private ConfigReader() {}.

Q: Can we access static variable through object reference?

A: Yes syntactically (obj.staticVar) but it's bad practice. Always use ClassName.staticVar — clearer intent.

Q: Difference between static import and normal import?

A: Normal import brings class name into scope. Static import brings static members — use assertEquals() instead of Assert.assertEquals().

Q: Can main method access non-static variables?

A: No directly — main is static, non-static variables belong to objects. Must create an object first: new MyClass().instanceMethod().

Q: How is static used in TestNG assertions?

A: All Assert methods are static: Assert.assertEquals(actual, expected). With static import: assertEquals(actual, expected).

Q: Static variable vs instance variable — memory?

A: Static: one copy in method area (metaspace). Instance: one copy per object on the heap. Static survives until class is unloaded.

Q: Can abstract class have static methods?

A: Yes — static methods in abstract classes are common for factory methods: public static LoginPage create(WebDriver driver).

Section 10: Interface vs Abstract Class

Interfaces and abstract classes are the foundation of framework architecture — BasePage, WebDriver, listener contracts, and report interfaces all rely on choosing the right abstraction.

Interface

A 100% contract (pre-Java 8). Defines what a class must do, not how. A class can implement multiple interfaces.

public interface IPageActions {
    void navigateTo();
    boolean isLoaded();
    String getPageTitle();
}

public interface IValidatable {
    void validateRequiredFields();
    List<String> getValidationErrors();
}

// Multiple interfaces
public class LoginPage implements IPageActions, IValidatable {
    @Override
    public void navigateTo() { /* ... */ }

    @Override
    public boolean isLoaded() { /* ... */ }

    @Override
    public String getPageTitle() { return driver.getTitle(); }

    @Override
    public void validateRequiredFields() { /* ... */ }

    @Override
    public List<String> getValidationErrors() { /* ... */ }
}

Default Methods (Java 8+)

Add behavior to interfaces without breaking existing implementations.

public interface IPageActions {
    void navigateTo();
    boolean isLoaded();

    // Default method — optional override
    default void waitForPageLoad() {
        new WebDriverWait(driver, Duration.ofSeconds(10))
            .until(d -> ((JavascriptExecutor) d)
                .executeScript("return document.readyState").equals("complete"));
    }
}

Static Methods in Interface

public interface BrowserUtils {
    static boolean isBrowserOpen(WebDriver driver) {
        try {
            driver.getTitle();
            return true;
        } catch (Exception e) {
            return false;
        }
    }
}

// Call without implementation
boolean open = BrowserUtils.isBrowserOpen(driver);

Functional Interface

@FunctionalInterface
public interface ElementAction {
    void perform(WebElement element);
    // Exactly ONE abstract method — enables lambdas

// Usage with lambda
ElementAction click = el -> el.click();
ElementAction getText = el -> System.out.println(el.getText());

elements.forEach(el -> click.perform(el));

  • IConfigurable — contract for config-driven components
  • IReportable — contract for test reporting listeners

Abstract Class

Partial implementation — mix of concrete methods (shared code) and abstract methods (force subclass implementation). Can have constructors, instance variables, any access modifier.

public abstract class BasePage {
    protected WebDriver driver;
    protected WebDriverWait wait;

    // Constructor — YES, abstract classes CAN have constructors
    protected BasePage(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
    }

    // Concrete method — shared by ALL pages
    public void click(By locator) {
        wait.until(ExpectedConditions.elementToBeClickable(locator)).click();
    }

    protected void sendKeys(By locator, String text) {
        wait.until(ExpectedConditions.visibilityOfElementLocated(locator)).sendKeys(text);
    }

    protected String getText(By locator) {
        return wait.until(ExpectedConditions.visibilityOfElementLocated(locator)).getText();
    }

    // Abstract method — force each page to implement
    public abstract boolean isLoaded();
    public abstract String getPageUrl();
}

Real Framework Decisions

  • BasePage → ABSTRACT CLASS — has shared code (click, sendKeys, wait) + forces isLoaded() implementation
  • IPageActions → INTERFACE — pure contract, no shared state, multiple unrelated classes can implement
  • WebDriver → INTERFACE — Selenium's core design; ChromeDriver, FirefoxDriver, RemoteWebDriver all implement it

  • ITestListener → INTERFACE — TestNG listener contract

Design Pattern: Interface + Abstract Class Combination

The most powerful framework pattern — interface defines the contract, abstract class provides shared implementation, concrete class completes it.

// 1. Interface — the contract
public interface IPageActions {
    void navigateTo();
    boolean isLoaded();
    void validatePage();
}

// 2. Abstract class — shared implementation
public abstract class BasePage implements IPageActions {
    protected WebDriver driver;
    protected WebDriverWait wait;

    protected BasePage(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
    }

    protected void click(By locator) {
        wait.until(ExpectedConditions.elementToBeClickable(locator)).click();
    }

    protected void type(By locator, String text) {
        WebElement el = wait.until(ExpectedConditions.visibilityOfElementLocated(locator));
        el.clear();
        el.sendKeys(text);
    }

    // Default implementation — can be overridden
    @Override
    public void validatePage() {
        if (!isLoaded()) {
            throw new PageNotFoundException(this.getClass().getSimpleName());
        }
    }

    // Force subclass to implement
    @Override
    public abstract boolean isLoaded();
}

// 3. Concrete class — page-specific
public class LoginPage extends BasePage {
    private By usernameField = By.id("username");
    private By passwordField = By.id("password");
    private By loginButton   = By.id("login-btn");

    public LoginPage(WebDriver driver) {
        super(driver);
    }

    @Override
    public void navigateTo() {
        driver.get(ConfigReader.getProperty("base.url") + "/login");
        validatePage();
    }

    @Override
    public boolean isLoaded() {
        return driver.findElement(loginButton).isDisplayed();
    }

    public void login(String username, String password) {
        type(usernameField, username);
        type(passwordField, password);
        click(loginButton);
    }
}

Interview Q&A — Section 10 (12+ Questions)

Q: Can an interface extend another interface?

A: Yes — interface inheritance is supported. interface IAdvancedActions extends IPageActions, IValidatable.

Q: Can an abstract class implement an interface?

A: Yes — and it can leave interface methods abstract. abstract class BasePage implements IPageActions — provides some implementations, leaves rest abstract.

Q: Can we create an object of an abstract class?

A: No — must subclass and implement all abstract methods. BasePage page = new BasePage(driver) is a compile error.

Q: Can an interface have a constructor?

A: No — interfaces cannot have constructors. Only abstract and concrete classes can.

Q: Why is WebDriver an interface and not an abstract class?

A: Selenium supports many browsers (Chrome, Firefox, Edge, Safari, Remote). Interface allows each browser driver to implement independently without sharing a common base class hierarchy. Also, WebDriver extends multiple interfaces: SearchContext, TakesScreenshot.

Q: What is a marker interface?

A: Empty interface with no methods — marks a capability. Serializable, Cloneable. In automation, you might create IScreenshotable as a marker for pages that support screenshot capture.

Q: Can an abstract class be final?

A: No — abstract and final are contradictory. Abstract means "must be extended"; final means "cannot be extended."

Q: Interface vs abstract class — Java 8 changes?

A: Java 8 added default and static methods to interfaces, blurring the line. But interfaces still can't have instance variables or constructors. Use abstract class when you need state (driver, wait).

Q: How does BasePage benefit from being abstract?

A: (1) Provides shared methods (click, type, wait) — DRY principle. (2) Forces each page to implement isLoaded() — compile-time enforcement. (3) Cannot be instantiated directly — must create specific pages.

Q: Can a concrete class extend an abstract class without implementing all abstract methods?

A: Only if the concrete class is also declared abstract. Otherwise, compile error — must implement all abstract methods.

Q: What is the diamond problem and how does Java handle it?

A: Multiple inheritance causes ambiguity if two interfaces have same default method. Java resolves by requiring explicit override: @Override public void method() { InterfaceA.super.method(); }.

Q: Why implement IPageActions interface if BasePage already has the methods?

A: Interface enables polymorphism — you can pass any IPageActions to a method without knowing the concrete page class. Enables generic page validation: void verifyPage(IPageActions page) { assertTrue(page.isLoaded()); }.

Q: Functional interface — why important in automation?

A: Enables lambda expressions for cleaner code: elements.forEach(el -> el.click()), custom wait conditions: wait.until(driver -> driver.findElement(By.id("x")).isDisplayed()), and callback patterns in retry utilities.
