Section 15: Selenium Fundamentals

What is Selenium?

Selenium is an open-source suite of tools for automating web browsers. It enables testers and developers to write scripts that interact with web applications the same way a real user would — clicking buttons, filling forms, navigating pages, and validating content.

Key characteristics: Cross-browser (Chrome, Firefox, Edge, Safari), cross-platform (Windows, macOS, Linux), multi-language (Java, Python, C#, JavaScript, Ruby), and integrates with TestNG, JUnit, Cucumber, and CI/CD pipelines.

Selenium Components — Evolution

1. Selenium RC (Remote Control) — Legacy / Deprecated

Selenium RC (2004–2016) used a client-server architecture. Test scripts sent commands to a Selenium Server, which injected JavaScript into the browser to simulate user actions. It was slow, fragile, and required the Selenium Server to be running separately.

  • Why deprecated: JavaScript injection caused cross-origin issues, was slow, and could not interact with native browser dialogs reliably. Replaced entirely by WebDriver in Selenium 3.

2. Selenium IDE — Record & Playback

A browser extension (Chrome/Firefox) that records user interactions and exports them as test scripts. Useful for quick prototyping and learning, but not suitable for complex, maintainable test suites.

3. Selenium WebDriver — Direct Browser Communication

The core automation API. WebDriver communicates directly with each browser's native driver (ChromeDriver, GeckoDriver, etc.) using a standardized protocol. This is what all modern Selenium tests use.

4. Selenium Grid — Distributed Execution

Runs tests in parallel across multiple machines, browsers, and OS combinations. Selenium 4 Grid is fully rewritten — supports Docker, Kubernetes, and event-driven architecture.

5. Selenium Manager (Selenium 4.6+) — Auto Driver Management

Built into Selenium 4.6+, Selenium Manager automatically downloads and manages the correct browser driver version. No need for WebDriverManager or manual driver setup in most cases.

Selenium Architecture

Selenium 3 Architecture

┌─────────────┐     JSON Wire Protocol      ┌─────────────────┐     Native Protocol     ┌──────────┐
│   Client    │ ──────────────────────────► │  Browser Driver │ ──────────────────────► │ Browser  │
│ (Java Test) │ ◄────────────────────────── │ (ChromeDriver)  │ ◄────────────────────── │ (Chrome) │

In Selenium 3, the client library communicated with the browser driver using the JSON Wire Protocol — a REST-based HTTP protocol where every action (click, sendKeys, findElement) was a separate HTTP request serialized as JSON.

Selenium 4 Architecture

┌─────────────┐     W3C WebDriver Protocol    ┌─────────────────┐     Native Protocol     ┌──────────┐
│   Client    │ ────────────────────────────► │  Browser Driver │ ──────────────────────► │ Browser  │
│ (Java Test) │ ◄──────────────────────────── │ (ChromeDriver)  │ ◄────────────────────── │ (Chrome) │
└─────────────┘     HTTP (W3C Standard)       └─────────────────┘     CDP / BiDi           └──────────┘

Selenium 4 adopted the W3C WebDriver Protocol as the official standard, replacing the JSON Wire Protocol. This ensures consistent behavior across all browser vendors and enables new features like CDP integration and relative locators.

Interview Q&A — Selenium Fundamentals

Q1: What is Selenium and why is it used?

A: Selenium is an open-source web automation framework. It is used to automate browser interactions for functional testing, regression testing, and cross-browser validation without manual effort.

Q2: What are the components of Selenium?

A: Selenium IDE (record/playback), Selenium WebDriver (core API), Selenium Grid (distributed execution), and Selenium Manager (auto driver management in Selenium 4).

Q3: What is the difference between Selenium 3 and Selenium 4?

A: Selenium 4 uses W3C WebDriver Protocol instead of JSON Wire Protocol, adds relative locators, CDP support, element screenshots, newWindow() API, built-in Selenium Manager, and a rewritten Grid.

Q4: What is JSON Wire Protocol?

A: The HTTP-based REST protocol used in Selenium 3 where each WebDriver command was serialized as JSON and sent over HTTP to the browser driver. Deprecated in favor of W3C WebDriver Protocol.

Q5: What is W3C WebDriver Protocol?

A: An official W3C standard that defines how automation clients communicate with browser drivers. Ensures consistent behavior across all browser vendors and is the protocol used in Selenium 4.

Q6: What is Selenium Grid used for?

A: Selenium Grid enables parallel test execution across multiple machines, browsers, and OS combinations. Tests are routed to available nodes, reducing total execution time.

Q7: What is Selenium Manager?

A: A built-in tool in Selenium 4.6+ that automatically detects the installed browser version and downloads the matching driver. Eliminates manual driver management.

Q8: Explain the WebDriver interface hierarchy.

A: SearchContext → WebDriver → RemoteWebDriver → Browser-specific drivers (ChromeDriver, FirefoxDriver, EdgeDriver). SearchContext provides findElement/findElements; WebDriver adds navigation and management APIs.

Q9: Is Selenium a testing tool or a library?

A: Selenium is a library/framework, not a standalone testing tool. It must be integrated with testing frameworks like TestNG or JUnit to create, organize, and report tests.

Q10: What is the difference between Selenium RC and WebDriver?

A: RC injected JavaScript into the browser (slow, fragile). WebDriver communicates directly with the browser's native automation API (fast, reliable). RC is fully deprecated.

Q11: What are relative locators in Selenium 4?

A: Methods to locate elements based on their spatial relationship to other elements: above(), below(), toLeftOf(), toRightOf(), and near(). Useful when elements lack unique IDs.

Q12: What is CDP in Selenium 4?

A: Chrome DevTools Protocol integration allows accessing browser internals — network interception, geolocation mocking, performance metrics, and console log capture.

Q13: Can Selenium automate mobile apps?

A: Selenium itself only automates web browsers. Mobile app automation is done via Appium, which extends the WebDriver protocol for iOS and Android.

Q14: What is the difference between Selenium and WebDriver?

A: Selenium is the overall project/suite. WebDriver is the specific API within Selenium that provides browser automation capabilities. "Selenium WebDriver" refers to using the WebDriver API as part of the Selenium suite.

Q15: Why was DesiredCapabilities deprecated?

A: DesiredCapabilities was a generic, untyped way to configure browsers. Selenium 4 replaced it with type-safe BrowserOptions classes (ChromeOptions, FirefoxOptions, EdgeOptions) that provide better IDE support and compile-time safety.

Q16: What is BiDi in Selenium 4?

A: Bidirectional WebDriver Protocol enables real-time, event-driven communication between the test script and browser — useful for listening to console logs, network events, and DOM mutations as they happen.

Q17: How does Selenium compare to Playwright?

A: Playwright has built-in auto-wait, faster execution via direct CDP/BiDi, and native parallel support. Selenium has a larger ecosystem, broader language support, and is the industry standard for enterprise Java teams.

Q18: What is the role of browser drivers?

A: Browser drivers (ChromeDriver, GeckoDriver) are executables that act as a bridge between Selenium client code and the actual browser. They translate WebDriver commands into browser-native actions.

Interview Q&A — Browser Automation

Q1: How do you launch Chrome browser in Selenium 4?

A: Simply instantiate ChromeDriver — Selenium Manager handles driver download: WebDriver driver = new ChromeDriver();

Q2: What is the difference between get() and navigate().to()?

A: Both load a URL and wait for page load. navigate().to() is part of the Navigation interface which also provides back(), forward(), and refresh().

Q3: How do you run tests in headless mode?

A: Add options.addArguments("--headless=new") to ChromeOptions before creating ChromeDriver.

Q4: What is ChromeOptions used for?

A: ChromeOptions configures Chrome browser behavior — headless mode, notifications, incognito, custom profiles, extensions, window size, and more.

Q5: How do you maximize the browser window?

A: driver.manage().window().maximize() or add --start-maximized to ChromeOptions.

Q6: How do you delete all cookies?

A: driver.manage().deleteAllCookies() — useful for ensuring clean test state between tests.

Q7: What are the three types of timeouts in Selenium?

A: Implicit wait (findElement polling), page load timeout (page load completion), and script timeout (async JavaScript execution).

Q8: How do you set a custom browser window size?

A: driver.manage().window().setSize(new Dimension(1280, 720))

Q9: What is the difference between close() and quit()?

A: close() closes the current browser window. quit() closes all windows and terminates the WebDriver session, releasing all resources.

Q10: How do you configure Firefox with a custom profile?

A: Use ProfilesIni to load a profile, set preferences on FirefoxProfile, then pass it to FirefoxOptions via setProfile().

Q11: When should you use headless mode?

A: CI/CD pipelines, Docker containers, server environments without GUI, and when speed is prioritized over visual debugging.

Q12: How do you disable browser notifications in Chrome?

A: options.addArguments("--disable-notifications") in ChromeOptions.

1. ID Locator

WebElement email = driver.findElement(By.id("email"));
  • Pros: Fastest locator, unique by HTML spec, most reliable
  • Cons: Not all elements have IDs; dynamic IDs (React/Vue) change on reload
  • When to use: Elements with stable, unique id attributes

2. Name Locator

WebElement username = driver.findElement(By.name("username"));
  • Pros: Fast, commonly available on form fields
  • Cons: May not be unique; multiple elements can share the same name
  • When to use: Input fields, radio buttons, checkboxes in forms

3. ClassName Locator

WebElement btn = driver.findElement(By.className("btn-primary"));
  • Pros: Simple syntax for styled elements
  • Cons: Returns only the first match; compound classes need CSS/XPath; not unique
  • When to use: When class is unique and you need the first matching element

4. TagName Locator

List<WebElement> inputs = driver.findElements(By.tagName("input"));
  • Pros: Finds all elements of a type
  • Cons: Too generic; returns many elements; rarely unique
  • When to use: Counting elements, iterating all links/images

5. LinkText Locator

WebElement signIn = driver.findElement(By.linkText("Sign In"));
  • Pros: Exact match, human-readable, fast
  • Cons: Case-sensitive; must match entire visible text; only works on <a> tags
  • When to use: Navigation links with exact, stable text

6. PartialLinkText Locator

WebElement link = driver.findElement(By.partialLinkText("Sign"));
  • Pros: Flexible partial matching
  • Cons: May match multiple links; less precise
  • When to use: Long link text where only part is stable

8. XPath — Comprehensive Guide

Types of XPath

// ABSOLUTE XPath — AVOID! Brittle, breaks with any DOM change
driver.findElement(By.xpath("/html/body/div[1]/form/input[2]"));

// RELATIVE XPath — Preferred
driver.findElement(By.xpath("//input[@id='email']"));

Attribute-based XPath

// Exact attribute match
driver.findElement(By.xpath("//input[@id='email']"));
driver.findElement(By.xpath("//button[@type='submit']"));

// Multiple attributes (AND)
driver.findElement(By.xpath("//input[@type='text' and @name='email']"));

// Multiple attributes (OR)
driver.findElement(By.xpath("//input[@id='email' or @name='email']"));

// Contains
driver.findElement(By.xpath("//div[contains(@class,'partial')]"));
driver.findElement(By.xpath("//input[contains(@id,'user')]"));

// Starts-with
driver.findElement(By.xpath("//input[starts-with(@id,'user')]"));

// Ends-with (XPath 2.0 — limited browser support, use contains instead)
driver.findElement(By.xpath("//input[substring(@id, string-length(@id) - 2) = 'end']"));

Text-based XPath

// Exact text
driver.findElement(By.xpath("//button[text()='Submit']"));
driver.findElement(By.xpath("//a[text()='Sign In']"));

// Contains text
driver.findElement(By.xpath("//span[contains(text(),'Welcome')]"));
driver.findElement(By.xpath("//div[contains(text(),'Error')]"));

// Normalize space (handles extra whitespace)
driver.findElement(By.xpath("//button[normalize-space()='Submit']"));
driver.findElement(By.xpath("//label[normalize-space(text())='Email Address']"));

XPath Axes

// Parent
driver.findElement(By.xpath("//input[@id='email']/parent::div"));
driver.findElement(By.xpath("//input[@id='email']/.."));

// Child
driver.findElement(By.xpath("//div[@class='form']/child::input"));
driver.findElement(By.xpath("//div[@class='form']/input"));

// Ancestor
driver.findElement(By.xpath("//input[@id='email']/ancestor::form"));
driver.findElement(By.xpath("//span/ancestor::div[@class='container']"));

// Descendant
driver.findElement(By.xpath("//div[@id='form']/descendant::input"));
driver.findElement(By.xpath("//div[@id='form']//input"));

// Following sibling
driver.findElement(By.xpath("//label[@for='email']/following-sibling::input"));

// Preceding sibling
driver.findElement(By.xpath("//input[@id='email']/preceding-sibling::label"));

Indexed XPath

// Select the 2nd input of type text
driver.findElement(By.xpath("(//input[@type='text'])[2]"));

// Last element
driver.findElement(By.xpath("(//div[@class='item'])[last()]"));

Dynamic Element Strategies

// Strategy 1: Partial attribute match (dynamic IDs)
driver.findElement(By.cssSelector("[id*='user-email']"));
driver.findElement(By.xpath("//input[contains(@id,'user-email')]"));

// Strategy 2: Parent-child relationship
driver.findElement(By.cssSelector("div.login-form > input[type='email']"));
driver.findElement(By.xpath("//div[@class='login-form']//input[@type='email']"));

// Strategy 3: Sibling relationship
driver.findElement(By.xpath("//label[text()='Email']/following-sibling::input"));

// Strategy 4: Index-based (last resort)
driver.findElement(By.xpath("(//input[@type='text'])[1]"));

// Strategy 5: data-testid attribute (best practice for test automation)
driver.findElement(By.cssSelector("[data-testid='login-email']"));

Locator Best Practices

  1. Prefer ID when available and stable — fastest and most reliable.
  2. Use CSS over XPath for performance when text search is not needed.
  3. Avoid absolute XPath — breaks with any DOM structure change.
  4. Use data-testid attributes — designed for automation, stable across UI changes.
  5. Never use dynamic IDs directly — use partial match or parent-child strategy.
  6. Keep locators short and readable — if a locator is 200 characters, refactor the HTML or use a better strategy.
  7. Avoid indexing when possible — (//input)[3] breaks when order changes.
  8. Store locators in Page Object classes — single point of maintenance.
  9. Test locators in browser DevTools — use document.querySelector() or $x() in console.
  10. Use unique, descriptive locator names in Page Objects — EMAIL_INPUT, not INPUT1.

Interview Q&A — Locators

Q1: What are the 8 locator strategies in Selenium?

A: ID, Name, ClassName, TagName, LinkText, PartialLinkText, CSS Selector, and XPath.

Q2: Which locator is fastest?

A: ID is the fastest because browsers index elements by ID internally. CSS Selector is the second fastest.

Q3: What is the difference between CSS Selector and XPath?

A: CSS is faster and forward-only. XPath is slower but supports bidirectional traversal and text-based searches.

Q4: When would you use XPath over CSS?

A: When you need to find elements by visible text, traverse up to parent/ancestor elements, or navigate between siblings.

Q5: What is the difference between findElement and findElements?

A: findElement returns a single WebElement (throws exception if not found). findElements returns a List (empty list if none found).

Q6: How do you handle dynamic IDs?

A: Use partial attribute matching: CSS [id*='partial'] or XPath contains(@id,'partial'). Alternatively, use parent-child or sibling relationships.

Q7: Why should you avoid absolute XPath?

A: Absolute XPath (/html/body/div[1]/...) breaks whenever any element is added or removed from the DOM hierarchy above the target element.

Q8: What are relative locators in Selenium 4?

A: above(), below(), toLeftOf(), toRightOf(), and near() — locate elements based on spatial relationship to a reference element.

Q9: How do you locate an element by its text?

A: XPath only: //button[text()='Submit'] or //span[contains(text(),'Welcome')]. CSS cannot search by text.

Q10: What is normalize-space() in XPath?

A: Strips leading/trailing whitespace and collapses internal whitespace. Useful when element text has inconsistent spacing: //button[normalize-space()='Submit']

Q11: How do you select the nth element when multiple match?

A: CSS: li:nth-child(3). XPath: (//li)[3]. Index is 1-based in both.

Q12: What is a data-testid attribute and why use it?

A: A custom HTML attribute added specifically for test automation. Stable across UI refactors since it is not tied to styling or behavior.

Q13: Can ClassName locator find multiple classes?

A: No. By.className("btn-primary") only accepts a single class. For compound classes, use CSS: .btn.btn-primary.

Q14: How do you locate elements inside a shadow DOM?

A: Standard locators cannot pierce shadow DOM. Use Selenium 4's getShadowRoot() or JavaScript execution to access shadow DOM elements.

Q15: What is the difference between child (>) and descendant (space) in CSS?

A: Child (>) selects direct children only. Descendant (space) selects all nested descendants at any depth.

Q16: How do you write an XPath for a button with text "Submit" inside a form?

A: //form//button[text()='Submit'] or //form/button[normalize-space()='Submit']

Q17: What locator strategy do you use for a table row by cell content?

A: XPath: //td[text()='John']/parent::tr or //tr[td[text()='John']]

Q18: How do you verify a locator finds exactly one element?

A: Use findElements and assert size: assert driver.findElements(by).size() == 1; Or test in browser DevTools console.

Q19: What is the following-sibling axis in XPath?

A: Selects all sibling elements that appear after the current element: //label[@for='email']/following-sibling::input

Q20: CSS selector for input with type text AND name email?

A: input[type='text'][name='email']

Q21: How do you locate a disabled button?

A: CSS: button[disabled] or button:disabled. XPath: //button[@disabled]

Q22: What is the best locator strategy priority order?

A: ID → Name → CSS Selector → XPath → LinkText → PartialLinkText → ClassName → TagName

Interview Q&A — WebElement Commands

Q1: What is a WebElement in Selenium?

A: A WebElement represents a single DOM element on the web page. It provides methods to interact with (click, sendKeys) and query (getText, getAttribute) the element.

Q2: What is the difference between getText() and getAttribute("value")?

A: getText() returns the visible text content between tags (for labels, divs, spans). getAttribute("value") returns the value attribute of input fields.

Q3: How do you upload a file in Selenium?

A: Find the file input element and use sendKeys with the absolute file path: fileInput.sendKeys("C:/path/to/file.pdf")

Q4: What is the difference between isDisplayed() and isEnabled()?

A: isDisplayed() checks if the element is visible on the page. isEnabled() checks if the element is interactive (not disabled).

Q5: Can you call findElement on a WebElement?

A: Yes. WebElement extends SearchContext, so you can chain locators to scope searches within a parent element.

Q6: What does isSelected() check?

A: Whether a checkbox, radio button, or option element is currently selected/checked.

Q7: What is getRect() in Selenium 4?

A: Returns a Rectangle object containing both the element's position (x, y) and dimensions (width, height) in a single call.

Q8: What is the difference between getAttribute() and getDomProperty()?

A: getAttribute() returns the HTML attribute as written in the source. getDomProperty() returns the live JavaScript property value, which may differ (e.g., after user input).

Q9: How do you get all links on a page?

A: List<WebElement> links = driver.findElements(By.tagName("a"));

Q10: What happens if you call click() on a hidden element?

A: Selenium throws ElementNotInteractableException because the element is not visible/clickable on the page.

Q11: How do you get the CSS color of an element?

A: element.getCssValue("color") returns the computed CSS property value as a string (e.g., "rgba(255, 0, 0, 1)").

Q12: What is getAccessibleName() used for?

A: Returns the accessible name of an element for accessibility testing — either the aria-label or the associated label text.

Section 19: Waits

Critical Topic: Waits are in the top 3 most-asked Selenium interview topics. Understanding implicit, explicit, and fluent waits — and when to use each — is essential.

Why Waits?

Modern web applications are dynamic — content loads via AJAX, animations play, elements appear/disappear. Without waits, tests fail with NoSuchElementException or ElementNotInteractableException because Selenium tries to interact before the element is ready.

Thread.sleep() — Anti-Pattern

// NEVER DO THIS in production tests
Thread.sleep(5000);  // Hard-coded 5 second pause
  • Wastes time: Always waits the full duration even if element is ready sooner.
  • Flaky: May not wait long enough on slow networks.
  • Not scalable: Cannot adjust dynamically to page load conditions.

Implicit Wait

driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

How it works: When findElement() is called, if the element is not immediately found, Selenium polls the DOM repeatedly (default every 500ms) until the element appears or the timeout expires.

  • Set once per driver session — applies to all subsequent findElement/findElements calls.
  • Pros: Simple, global, no extra code per element.
  • Cons: Applies same timeout to all elements; cannot wait for specific conditions (clickable, visible); slows down tests when elements don't exist.

Explicit Wait

import org.openqa.selenium.support.ui.WebDriverWait;
import org.openqa.selenium.support.ui.ExpectedConditions;
import java.time.Duration;

WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
WebElement element = wait.until(
    ExpectedConditions.visibilityOfElementLocated(By.id("dashboard"))
);

WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));

// Element present in DOM (may not be visible)
wait.until(ExpectedConditions.presenceOfElementLocated(By.id("hidden-div")));

// Specific text appears in element

    By.id("status"), "Success"));

wait.until(ExpectedConditions.titleIs("My App - Dashboard"));

// Alert appears
wait.until(ExpectedConditions.alertIsPresent());

// Frame available and switch to it
wait.until(ExpectedConditions.frameToBeAvailableAndSwitchToIt("iframe-id"));

// Number of browser windows
wait.until(ExpectedConditions.numberOfWindowsToBe(2));

// Element becomes stale (removed from DOM)
wait.until(ExpectedConditions.stalenessOf(oldElement));

// URL checks

wait.until(ExpectedConditions.urlToBe("https://app.example.com/dashboard"));

    WebElement el = driver.findElement(By.id("counter"));
    return Integer.parseInt(el.getText()) > 0;
});

Fluent Wait

import org.openqa.selenium.support.ui.FluentWait;
import org.openqa.selenium.NoSuchElementException;

FluentWait<WebDriver> fluentWait = new FluentWait<>(driver)
    .withTimeout(Duration.ofSeconds(30))

    .ignoring(NoSuchElementException.class)
    .ignoring(StaleElementReferenceException.class);

    driver.findElement(By.id("dynamic-content"))

When to use: Irregular load times, animations, elements that may throw transient exceptions during loading.

WaitUtils Utility Class

import org.openqa.selenium.*;
import org.openqa.selenium.support.ui.*;
import java.time.Duration;

public class WaitUtils {
    private WebDriver driver;
    private WebDriverWait wait;

    public WaitUtils(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(15));
    }

    public WebElement waitForVisible(By locator) {
        return wait.until(ExpectedConditions.visibilityOfElementLocated(locator));
    }

    public WebElement waitForClickable(By locator) {
        return wait.until(ExpectedConditions.elementToBeClickable(locator));
    }

    public WebElement waitForPresence(By locator) {
        return wait.until(ExpectedConditions.presenceOfElementLocated(locator));
    }

    public boolean waitForInvisible(By locator) {
        return wait.until(ExpectedConditions.invisibilityOfElementLocated(locator));
    }

    public boolean waitForText(By locator, String text) {
        return wait.until(ExpectedConditions.textToBePresentInElementLocated(locator, text));
    }

    public Alert waitForAlert() {
        return wait.until(ExpectedConditions.alertIsPresent());
    }

    public void waitForUrlContains(String partialUrl) {
        wait.until(ExpectedConditions.urlContains(partialUrl));
    }

    public void waitForTitleContains(String title) {
        wait.until(ExpectedConditions.titleContains(title));
    }

    public void waitForFrameAndSwitch(String frameId) {
        wait.until(ExpectedConditions.frameToBeAvailableAndSwitchToIt(frameId));
    }

    public void waitForWindowCount(int count) {
        wait.until(ExpectedConditions.numberOfWindowsToBe(count));
    }
}

Interview Q&A — Waits

Q1: What are the types of waits in Selenium?

A: Implicit wait, explicit wait (WebDriverWait), and fluent wait (FluentWait). Thread.sleep() is not a Selenium wait — it is a Java pause and should be avoided.

Q2: What is the difference between implicit and explicit wait?

A: Implicit wait is set globally and polls for element presence on every findElement call. Explicit wait waits for a specific condition on a specific element with a defined timeout.

Q3: Why should you never use Thread.sleep()?

A: It wastes time (always waits full duration), makes tests flaky (may not wait long enough), and cannot adapt to varying load times.

Q4: Why should you never mix implicit and explicit waits?

A: Their timeouts add together unpredictably. An explicit wait of 10s with implicit wait of 10s could take up to 20s total, causing tests to hang.

Q5: What is the difference between visibilityOfElementLocated and presenceOfElementLocated?

A: presenceOfElementLocated checks if element exists in DOM (may be hidden). visibilityOfElementLocated checks if element exists AND is visible (not hidden via CSS).

Q6: What is elementToBeClickable?

A: Waits until the element is both visible and enabled — ready to receive a click event. Most reliable condition before clicking.

Q7: What is stalenessOf in ExpectedConditions?

A: Waits until a previously located element is removed from the DOM. Useful after actions that refresh part of the page (e.g., AJAX table refresh).

Q8: What is FluentWait?

A: A configurable wait with custom polling interval and ability to ignore specific exceptions during polling. More flexible than WebDriverWait.

Q9: What is the default polling frequency for WebDriverWait?

A: 500 milliseconds — Selenium checks the condition every 500ms until timeout or condition is met.

Q10: How do you wait for a page to load completely?

A: Set pageLoadTimeout: driver.manage().timeouts().pageLoadTimeout(Duration.ofSeconds(30)). Or use explicit wait on a known element that appears after load.

Q11: What is a custom ExpectedCondition?

A: A lambda expression passed to wait.until() that returns true when the custom condition is met: wait.until(driver -> driver.findElements(by).size() > 5)

Q12: What happens when an explicit wait times out?

A: Selenium throws TimeoutException. The test should catch this or let the framework report the failure.

Q13: Should implicit wait be set to zero?

A: Best practice when using explicit waits: set implicit wait to 0 or a very low value (1-2s) to avoid timeout stacking.

Q14: How do you wait for an element's text to change?

A: Use textToBePresentInElementLocated or a custom condition: wait.until(ExpectedConditions.textToBePresentInElementLocated(by, "Expected Text"))

Q15: What is the recommended wait strategy for a test framework?

A: Set implicit wait to 0. Use explicit waits via a WaitUtils utility class for all element interactions. Use fluent wait only for special cases with irregular timing.

Q16: How do you wait for an AJAX call to complete?

A: Wait for a loading spinner to disappear: wait.until(ExpectedConditions.invisibilityOfElementLocated(By.id("spinner"))) or wait for the result element to appear.

Q17: What is scriptTimeout used for?

A: Sets the maximum time Selenium waits for asynchronous JavaScript (executeAsyncScript) to complete before throwing TimeoutException.

Complete Example — All Dropdown Types

import org.openqa.selenium.*;
import org.openqa.selenium.support.ui.*;
import java.time.Duration;
import java.util.List;

public class DropdownHandler {
    WebDriver driver;
    WebDriverWait wait;

    public DropdownHandler(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
    }

    // Standard HTML select dropdown
    public void selectByText(By locator, String text) {
        Select select = new Select(driver.findElement(locator));
        select.selectByVisibleText(text);
    }

    public void selectByValue(By locator, String value) {
        Select select = new Select(driver.findElement(locator));
        select.selectByValue(value);
    }

    // Custom/Bootstrap dropdown
    public void selectCustomOption(By dropdownLocator, String optionText) {
        driver.findElement(dropdownLocator).click();
        By optionLocator = By.xpath(
            "//ul[contains(@class,'dropdown')]//li[normalize-space()='" + optionText + "']"
        );
        wait.until(ExpectedConditions.elementToBeClickable(optionLocator)).click();
    }

    // Auto-suggest dropdown
    public void selectAutoSuggest(By inputLocator, String partialText, String fullText) {
        WebElement input = driver.findElement(inputLocator);
        input.clear();
        input.sendKeys(partialText);
        By suggestion = By.xpath(
            "//ul[contains(@class,'suggest')]//li[contains(text(),'" + fullText + "')]"
        );
        wait.until(ExpectedConditions.elementToBeClickable(suggestion)).click();
    }

    // Get all options from standard select
    public List<String> getAllOptions(By locator) {
        Select select = new Select(driver.findElement(locator));
        return select.getOptions().stream()
            .map(WebElement::getText)
            .toList();
    }
}

Interview Q&A — Dropdowns

Q1: How do you handle dropdowns in Selenium?

A: For HTML <select> elements, use the Select class. For custom/Bootstrap dropdowns, click to open and then click the desired option.

Q2: What are the three ways to select an option in a dropdown?

A: selectByVisibleText(), selectByValue(), and selectByIndex().

Q3: Can you use Select class on a div-based dropdown?

A: No. Select class only works with HTML <select> elements. Div/ul-based dropdowns require click-based interaction.

Q4: How do you handle multi-select dropdowns?

A: Check isMultiple(), then call selectByVisibleText() multiple times for each option. Use getAllSelectedOptions() to verify.

Q5: How do you get all options from a dropdown?

A: dropdown.getOptions() returns a List<WebElement> of all options.

Q6: What happens if you call deselectAll() on a single-select dropdown?

A: Selenium throws UnsupportedOperationException because single-select dropdowns cannot be deselected.

Q7: How do you handle auto-suggest dropdowns?

A: Type partial text with sendKeys(), wait for suggestions to appear, then click the matching suggestion element.

Q8: How do you verify the selected option in a dropdown?

A: dropdown.getFirstSelectedOption().getText() for single-select, or getAllSelectedOptions() for multi-select.

Q9: What is the difference between selectByVisibleText and selectByValue?

A: selectByVisibleText matches the displayed text between <option> tags. selectByValue matches the value attribute of the <option> element.

Q10: How do you select an option that is not visible (needs scrolling)?

A: Use selectByValue() or selectByIndex() which don't require the option to be visible. Or scroll the dropdown first using JavaScript.

Complete Alert Handling Examples

import org.openqa.selenium.*;
import org.openqa.selenium.support.ui.*;
import java.time.Duration;

public class AlertHandler {
    WebDriver driver;
    WebDriverWait wait;

    public AlertHandler(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
    }

    // Handle simple alert — click OK
    public String handleSimpleAlert() {
        Alert alert = wait.until(ExpectedConditions.alertIsPresent());
        String text = alert.getText();
        alert.accept();
        return text;
    }

    // Handle confirmation alert — click OK or Cancel
    public void handleConfirmationAlert(boolean accept) {
        Alert alert = wait.until(ExpectedConditions.alertIsPresent());
        if (accept) {
            alert.accept();
        } else {
            alert.dismiss();
        }
    }

    // Handle prompt alert — enter text and submit
    public void handlePromptAlert(String inputText) {
        Alert alert = wait.until(ExpectedConditions.alertIsPresent());
        alert.sendKeys(inputText);
        alert.accept();
    }
}

Sweet Alert / Bootstrap Modals — Not Real Alerts

Important: SweetAlert, Bootstrap modals, and custom popup dialogs are NOT JavaScript alerts. They are regular HTML elements in the DOM. Handle them with normal locators — do NOT use switchTo().alert().

// SweetAlert / Bootstrap modal — use regular locators
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(5));

// Wait for modal to appear
WebElement modal = wait.until(ExpectedConditions.visibilityOfElementLocated(
    By.cssSelector(".swal2-popup, .modal-dialog")
));

// Read modal text
String modalText = modal.findElement(By.cssSelector(".swal2-content, .modal-body")).getText();

// Click OK/Confirm button
modal.findElement(By.cssSelector(".swal2-confirm, .btn-primary")).click();

Interview Q&A — Alerts

Q1: How do you handle alerts in Selenium?

A: Switch to the alert using driver.switchTo().alert(), then use accept(), dismiss(), getText(), or sendKeys() as needed.

Q2: What are the three types of JavaScript alerts?

A: Simple alert (OK only), confirmation alert (OK + Cancel), and prompt alert (text input + OK + Cancel).

Q3: What is the difference between accept() and dismiss()?

A: accept() clicks OK. dismiss() clicks Cancel. dismiss() only works on confirmation and prompt alerts.

Q4: How do you enter text in a prompt alert?

A: alert.sendKeys("text") followed by alert.accept().

Q5: What is UnhandledAlertException?

A: Thrown when an unexpected alert appears while Selenium is trying to interact with the page. Handle by switching to alert and dismissing it.

Q6: Can you use switchTo().alert() for Bootstrap modals?

A: No. Bootstrap modals and SweetAlert are HTML elements, not JavaScript alerts. Use regular findElement locators to interact with them.

Q7: How do you wait for an alert to appear?

A: wait.until(ExpectedConditions.alertIsPresent())

Q8: How do you handle HTTP authentication popups?

A: Embed credentials in the URL: https://username:password@example.com. Or use Selenium 4 CDP to handle authentication.

Q9: What happens if you try to find an element while an alert is open?

A: Selenium throws UnhandledAlertException because the alert blocks interaction with the page DOM.

Q10: How do you verify alert text in a test?

A: String alertText = driver.switchTo().alert().getText(); then assert against expected text before accepting/dismissing.

Section 22: Frames & Windows

Frames (iFrames)

A frame (iframe) is an embedded HTML document within another HTML page. Selenium can only interact with elements in the currently focused frame — you must switch to a frame before locating elements inside it.

Switching to Frames

// By index (0-based)
driver.switchTo().frame(0);   // First frame on page
driver.switchTo().frame(1);   // Second frame

// By name or ID attribute
driver.switchTo().frame("frameName");
driver.switchTo().frame("iframe-id");

// By WebElement reference
WebElement frameElement = driver.findElement(By.id("my-frame"));
driver.switchTo().frame(frameElement);

Switching Back

// Return to main page content (top-level)
driver.switchTo().defaultContent();

// Return to parent frame (one level up)
driver.switchTo().parentFrame();

├── iframe#header (frame 0)

│   └── iframe#inner-form (nested frame)
│       └── <input id="email">  ← Must switch: defaultContent → frame 1 → inner-form
└── iframe#footer (frame 2)

Complete Frame Handling Code

import org.openqa.selenium.*;
import org.openqa.selenium.support.ui.*;
import java.time.Duration;

public class FrameHandler {
    WebDriver driver;
    WebDriverWait wait;

    public FrameHandler(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
    }

    public void interactWithNestedFrame() {
        // Switch to outer frame
        wait.until(ExpectedConditions.frameToBeAvailableAndSwitchToIt("outer-frame"));

        // Switch to inner frame
        wait.until(ExpectedConditions.frameToBeAvailableAndSwitchToIt("inner-frame"));

        // Interact with element inside nested frame
        WebElement email = wait.until(
            ExpectedConditions.visibilityOfElementLocated(By.id("email"))
        );
        email.sendKeys("user@example.com");

        WebElement submit = driver.findElement(By.id("submit"));
        submit.click();

        // Navigate back up
        driver.switchTo().parentFrame();  // Back to outer frame
        driver.switchTo().defaultContent(); // Back to main page
    }

    public void switchToFrameByElement(By locator) {
        WebElement frame = wait.until(ExpectedConditions.presenceOfElementLocated(locator));
        driver.switchTo().frame(frame);
    }
}

Complete Multi-Window Handling

import org.openqa.selenium.*;
import org.openqa.selenium.support.ui.*;
import java.time.Duration;
import java.util.Set;

public class WindowHandler {
    WebDriver driver;
    WebDriverWait wait;

    public WindowHandler(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
    }

    public void handlePopupWindow(By triggerLocator) {
        String parentHandle = driver.getWindowHandle();

        // Click element that opens new window
        driver.findElement(triggerLocator).click();

        // Wait for new window to open
        wait.until(ExpectedConditions.numberOfWindowsToBe(2));

        // Switch to new window
        switchToNewWindow(parentHandle);

        // Perform actions in popup
        System.out.println("Popup title: " + driver.getTitle());
        driver.findElement(By.id("confirm-btn")).click();

        // Close popup and return to parent
        driver.close();
        driver.switchTo().window(parentHandle);
    }

    private void switchToNewWindow(String parentHandle) {
        Set<String> handles = driver.getWindowHandles();
        for (String handle : handles) {
            if (!handle.equals(parentHandle)) {
                driver.switchTo().window(handle);
                return;
            }
        }
        throw new NoSuchWindowException("New window not found");
    }

    public void closeAllExcept(String keepHandle) {
        for (String handle : driver.getWindowHandles()) {
            if (!handle.equals(keepHandle)) {
                driver.switchTo().window(handle);
                driver.close();
            }
        }
        driver.switchTo().window(keepHandle);
    }
}

Interview Q&A — Frames & Windows

Q1: What is an iframe and why must you switch to it?

A: An iframe embeds a separate HTML document inside a page. Selenium's findElement only searches the current context — you must switchTo().frame() to access elements inside.

Q2: What are the three ways to switch to a frame?

A: By index (int), by name/id (String), or by WebElement reference.

Q3: What is the difference between defaultContent() and parentFrame()?

A: defaultContent() returns to the top-level page. parentFrame() returns one level up to the immediate parent frame.

Q4: How do you handle nested frames?

A: Switch to outer frame first, then inner frame. To exit, use parentFrame() to go up one level, or defaultContent() to return to the main page.

Q5: What is a window handle?

A: A unique string identifier assigned by the browser to each window/tab. Used with switchTo().window(handle) to switch between windows.

Q6: What is the difference between close() and quit()?

A: close() closes the current window only. quit() closes all windows and terminates the WebDriver session.

Q7: How do you open a new tab in Selenium 4?

A: driver.switchTo().newWindow(WindowType.TAB) then navigate with driver.get().

Q8: How do you switch to a newly opened window?

A: Store the parent handle, wait for numberOfWindowsToBe(2), iterate getWindowHandles(), switch to the handle that is not the parent.

Q9: What happens if you try to find an element inside a frame without switching?

A: Selenium throws NoSuchElementException because it searches the current context (main page), not the frame content.

Q10: How do you wait for a frame to load before switching?

A: wait.until(ExpectedConditions.frameToBeAvailableAndSwitchToIt("frame-id"))

Q11: Can a page have multiple frames?

A: Yes. Use index, name/id, or WebElement to switch to the specific frame you need. Always switch back to defaultContent when done.

Q12: How do you close all popup windows and return to the main window?

A: Store main handle, iterate all handles, close each non-main window, then switchTo().window(mainHandle).

Q13: What is NoSuchWindowException?

A: Thrown when trying to switch to a window handle that no longer exists (already closed) or is invalid.

Q14: How do you get the total number of open windows?

A: driver.getWindowHandles().size()
