Section 23: Actions Class

What Is the Actions Class?

The Actions class in Selenium WebDriver provides a low-level, user-like way to perform advanced interactions that go beyond simple element.click() or sendKeys(). It simulates real mouse movements, keyboard input, and composite gestures — hover menus, drag-and-drop, sliders, right-click context menus, and multi-key combinations.

Key idea: Actions builds a queue of interaction steps. Nothing happens until you call .perform() (or .build().perform()).

Creating an Actions Object

import org.openqa.selenium.interactions.Actions;

Actions actions = new Actions(driver);

Mouse Actions

moveToElement() — Hover

Syntax: actions.moveToElement(element).perform();

WebElement menu = driver.findElement(By.id("products-menu"));
actions.moveToElement(menu).perform();

Real scenario: Hover over a navigation menu item so a submenu appears, then click a submenu link.

click() — Alternative Click

Syntax: actions.click(element).perform();

WebElement submitBtn = driver.findElement(By.id("submit"));
actions.click(submitBtn).perform();

Real scenario: Use when a standard click is intercepted by an overlay or another element.

doubleClick()

Syntax: actions.doubleClick(element).perform();

WebElement fileItem = driver.findElement(By.xpath("//span[text()='report.pdf']"));
actions.doubleClick(fileItem).perform();

Real scenario: Open a file in a file manager, or select a word in a text editor.

contextClick() — Right-Click

Syntax: actions.contextClick(element).perform();

WebElement row = driver.findElement(By.xpath("//tr[td='John Doe']"));
actions.contextClick(row).perform();

Real scenario: Open a context menu to copy, delete, or edit a table row.

clickAndHold() + release() — Slider / Drag

Syntax:

actions.clickAndHold(slider).moveByOffset(200, 0).release().perform();

Real scenario: Drag a price-range slider or resize a panel.

dragAndDrop(source, target)

Syntax: actions.dragAndDrop(source, target).perform();

WebElement source = driver.findElement(By.id("task-1"));
WebElement target = driver.findElement(By.id("done-column"));
actions.dragAndDrop(source, target).perform();

Real scenario: Move a Kanban card from "To Do" to "Done".

dragAndDropBy(source, xOffset, yOffset)

Syntax: actions.dragAndDropBy(source, xOffset, yOffset).perform();

WebElement handle = driver.findElement(By.className("slider-handle"));
actions.dragAndDropBy(handle, 150, 0).perform();

Real scenario: Pixel-precise drag when there is no explicit drop target element.

Keyboard Actions

sendKeys(Keys.ENTER) — Submit Form

WebElement searchBox = driver.findElement(By.name("q"));
actions.sendKeys(searchBox, "Selenium").sendKeys(Keys.ENTER).perform();

keyDown + click + keyUp — Multi-Select

WebElement item1 = driver.findElement(By.id("item-1"));
WebElement item2 = driver.findElement(By.id("item-2"));
actions.keyDown(Keys.CONTROL).click(item1).click(item2).keyUp(Keys.CONTROL).perform();

sendKeys(Keys.chord(...)) — Select All

WebElement textArea = driver.findElement(By.id("editor"));
actions.click(textArea).sendKeys(Keys.chord(Keys.CONTROL, "a")).perform();

Common Keys: TAB, ESCAPE, ARROW_DOWN

actions.sendKeys(Keys.TAB).perform();           // move focus
actions.sendKeys(Keys.ESCAPE).perform();        // close modal
actions.sendKeys(Keys.ARROW_DOWN).perform();    // navigate dropdown

Action Chains

Chain multiple steps into one user flow:

WebElement menu    = driver.findElement(By.id("products"));
WebElement submenu = driver.findElement(By.linkText("Laptops"));

actions.moveToElement(menu).click(submenu).perform();
Critical: You MUST call .perform() or .build().perform(). Without it, the queued actions never execute.

Selenium 4 New Scroll Actions

// Scroll element into view
actions.scrollToElement(element).perform();

// Scroll by pixel amount (x, y)
actions.scrollByAmount(0, 500).perform();

Complete Example — Menu Hover, Drag-Drop, Slider, Right-Click

import org.openqa.selenium.*;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.interactions.Actions;
import java.time.Duration;

public class ActionsDemo {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();
        driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
        driver.get("https://example.com/app");
        Actions actions = new Actions(driver);

        // 1. Hover menu → click submenu
        WebElement menu = driver.findElement(By.id("nav-products"));
        WebElement submenu = driver.findElement(By.linkText("Electronics"));
        actions.moveToElement(menu).pause(Duration.ofMillis(500))
               .click(submenu).perform();

        // 2. Drag and drop
        WebElement source = driver.findElement(By.id("drag-item"));
        WebElement target = driver.findElement(By.id("drop-zone"));
        actions.dragAndDrop(source, target).perform();

        // 3. Slider via clickAndHold
        WebElement slider = driver.findElement(By.className("slider-thumb"));
        actions.clickAndHold(slider).moveByOffset(200, 0).release().perform();

        // 4. Right-click context menu
        WebElement row = driver.findElement(By.xpath("//tr[td='Invoice-101']"));
        actions.contextClick(row).perform();

        driver.quit();
    }
}

Section 24: JavaScript Executor

What Is JavascriptExecutor?

JavascriptExecutor is a Selenium interface that lets you execute JavaScript directly in the browser's context. Use it when standard WebDriver commands cannot interact with an element — hidden inputs, read-only fields, elements blocked by overlays, or custom widgets.

Casting the Driver

JavascriptExecutor js = (JavascriptExecutor) driver;

Common Operations

Click Element (when regular click fails)

WebElement btn = driver.findElement(By.id("hidden-submit"));
js.executeScript("arguments[0].click();", btn);

Scroll to Element

WebElement footer = driver.findElement(By.id("footer"));
js.executeScript("arguments[0].scrollIntoView(true);", footer);

Scroll to Bottom of Page

js.executeScript("window.scrollTo(0, document.body.scrollHeight)");

Scroll by Pixels

js.executeScript("window.scrollBy(0, 500)");

Highlight Element (debugging)

js.executeScript("arguments[0].style.border='3px solid red'", element);

Get Text from Element

String text = (String) js.executeScript("return arguments[0].innerText", element);

Set Value (hidden inputs, read-only fields)

js.executeScript("arguments[0].value='admin@example.com'", element);

Get Page Title

String title = (String) js.executeScript("return document.title");

Get Domain

String domain = (String) js.executeScript("return document.domain");

Remove Attribute (e.g., readonly)

js.executeScript("arguments[0].removeAttribute('readonly')", dateField);

Trigger Change Event (after setting value)

js.executeScript(
    "arguments[0].value='2026-07-14';" +
    "arguments[0].dispatchEvent(new Event('change', { bubbles: true }));",
    dateField
);

Complete Utility Class — WebTableUtils

import org.openqa.selenium.*;
import org.openqa.selenium.support.ui.ExpectedConditions;
import org.openqa.selenium.support.ui.WebDriverWait;
import java.time.Duration;
import java.util.ArrayList;
import java.util.List;

public class WebTableUtils {
    private WebDriver driver;
    private WebDriverWait wait;
    private static final String TABLE_XPATH = "//table[@id='data-table']";

    public WebTableUtils(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
    }

    public int getRowCount() {
        wait.until(ExpectedConditions.presenceOfElementLocated(By.xpath(TABLE_XPATH)));
        return driver.findElements(By.xpath(TABLE_XPATH + "//tbody/tr")).size();
    }

    public int getColumnCount() {
        return driver.findElements(By.xpath(TABLE_XPATH + "//thead/tr/th")).size();
    }

    public String getCellValue(int row, int col) {
        String xpath = TABLE_XPATH + "//tbody/tr[" + row + "]/td[" + col + "]";
        return driver.findElement(By.xpath(xpath)).getText().trim();
    }

    public List<String> getColumnData(int colIndex) {
        List<String> columnData = new ArrayList<>();
        int rows = getRowCount();
        for (int i = 1; i <= rows; i++) {
            columnData.add(getCellValue(i, colIndex));
        }
        return columnData;
    }

    public int searchTable(String searchText) {
        List<WebElement> rows = driver.findElements(By.xpath(TABLE_XPATH + "//tbody/tr"));
        for (int i = 0; i < rows.size(); i++) {
            if (rows.get(i).getText().contains(searchText)) {
                return i + 1; // 1-based row number
            }
        }
        return -1;
    }

    public void clickActionInRow(String searchText, int actionColIndex) {
        int rowNum = searchTable(searchText);
        if (rowNum == -1) throw new NoSuchElementException("Row not found: " + searchText);
        String xpath = TABLE_XPATH + "//tbody/tr[" + rowNum + "]/td[" + actionColIndex + "]//button";
        driver.findElement(By.xpath(xpath)).click();
    }
}

Screenshot Utility Class

import org.openqa.selenium.*;
import org.apache.commons.io.FileUtils;
import java.io.File;
import java.io.IOException;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;

public class ScreenshotUtils {
    private static final String BASE_DIR = "screenshots/";

    public static String capture(WebDriver driver, String testName) {
        String timestamp = LocalDateTime.now()
            .format(DateTimeFormatter.ofPattern("yyyyMMdd_HHmmss"));
        String folder = BASE_DIR + testName + "/";
        new File(folder).mkdirs();
        String filePath = folder + testName + "_" + timestamp + ".png";

        try {
            File src = ((TakesScreenshot) driver).getScreenshotAs(OutputType.FILE);
            FileUtils.copyFile(src, new File(filePath));
        } catch (IOException e) {
            throw new RuntimeException("Screenshot failed: " + e.getMessage());
        }
        return filePath;
    }

    public static String captureElement(WebElement element, String testName) {
        String timestamp = LocalDateTime.now()
            .format(DateTimeFormatter.ofPattern("yyyyMMdd_HHmmss"));
        String filePath = BASE_DIR + testName + "/element_" + timestamp + ".png";
        new File(BASE_DIR + testName + "/").mkdirs();
        try {
            File src = element.getScreenshotAs(OutputType.FILE);
            FileUtils.copyFile(src, new File(filePath));
        } catch (IOException e) {
            throw new RuntimeException("Element screenshot failed", e);
        }
        return filePath;
    }
}

Section 27: Page Object Model (POM)

What Is POM?

The Page Object Model is a design pattern that separates page structure (locators and UI actions) from test logic (assertions and workflows). Each web page gets a dedicated Java class encapsulating its elements and interactions.

Why POM?

  • Maintainability: Locator changes happen in one place.
  • Reusability: Page methods shared across multiple tests.
  • Readability: Tests read like business scenarios.

POM Architecture

┌─────────────┐     calls      ┌──────────────┐     uses     ┌────────────┐     drives    ┌──────────┐
│  Test Class │ ──────────────▶│  Page Object │ ───────────▶│ WebElement │ ────────────▶│   Page   │
│ LoginTest   │                │  LoginPage   │             │  locators  │             │ Browser  │
└─────────────┘                └──────────────┘             └────────────┘             └──────────┘

POM Rules

  1. One page class per web page (or major view).
  2. All locators live inside the page class — never in tests.
  3. All page interactions are exposed as methods.
  4. Return type: next page object (navigation) or void (action).
  5. No assertions inside page objects — tests assert outcomes.

Without POM vs With POM

Without POM (anti-pattern)

@Test
public void loginTest() {
    driver.findElement(By.id("username")).sendKeys("admin");
    driver.findElement(By.id("password")).sendKeys("pass123");
    driver.findElement(By.id("loginBtn")).click();
    Assert.assertEquals(driver.getTitle(), "Home");
}

With POM (recommended)

@Test
public void loginTest() {
    HomePage home = new LoginPage(driver)
        .enterUsername("admin")
        .enterPassword("pass123")
        .clickLogin();
    Assert.assertTrue(home.verifyHomePage());
}

Complete POM Example

LoginPage.java

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.support.FindBy;
import org.openqa.selenium.support.PageFactory;

public class LoginPage {
    private WebDriver driver;

    @FindBy(id = "username") private WebElement usernameField;
    @FindBy(id = "password") private WebElement passwordField;
    @FindBy(id = "loginBtn")  private WebElement loginButton;

    public LoginPage(WebDriver driver) {
        this.driver = driver;
        PageFactory.initElements(driver, this);
    }

    public LoginPage enterUsername(String username) {
        usernameField.clear();
        usernameField.sendKeys(username);
        return this;
    }

    public LoginPage enterPassword(String password) {
        passwordField.clear();
        passwordField.sendKeys(password);
        return this;
    }

    public HomePage clickLogin() {
        loginButton.click();
        return new HomePage(driver);
    }

    public void login(String user, String pass) {
        enterUsername(user).enterPassword(pass).clickLogin();
    }
}

HomePage.java

public class HomePage {
    private WebDriver driver;

    @FindBy(className = "welcome-msg") private WebElement welcomeText;
    @FindBy(id = "dashboard")         private WebElement dashboard;

    public HomePage(WebDriver driver) {
        this.driver = driver;
        PageFactory.initElements(driver, this);
    }

    public boolean verifyHomePage() {
        return dashboard.isDisplayed();
    }

    public String getWelcomeText() {
        return welcomeText.getText();
    }
}

POM Best Practices (10 Points)

  1. One class per page or major UI section.
  2. Use meaningful method names reflecting user actions.
  3. Keep locators private; expose behavior, not elements.
  4. Return next page object after navigation.
  5. Never put assertions in page classes.
  6. Extend a BasePage for shared waits and utilities.
  7. Avoid hardcoded test data — pass via method parameters.
  8. Split large pages into component classes.
  9. Use explicit waits inside page methods.
  10. Keep page objects free of TestNG/JUnit annotations.

POM Anti-Patterns

Anti-PatternProblemFix
God Page ObjectOne class for entire appSplit by page/component
Assertions in pagesMixes UI logic with test logicAssert only in test classes
Hardcoded dataNot reusable across testsParameterize methods
Public WebElementsTests bypass page methodsKeep elements private
Driver in testsTests interact with DOM directlyAll interaction via page objects

Section 28: Page Factory

What Is Page Factory?

Page Factory is Selenium's built-in implementation of POM using @FindBy annotations. It initializes WebElements lazily via PageFactory.initElements(), reducing boilerplate locator code.

@FindBy Annotation

@FindBy(id = "email")
WebElement emailField;

Other @FindBy Locator Strategies

StrategyExample
id@FindBy(id = "submit")
name@FindBy(name = "username")
className@FindBy(className = "btn-primary")
css@FindBy(css = "input[type='email']")
xpath@FindBy(xpath = "//button[text()='Login']")
linkText@FindBy(linkText = "Forgot Password")
partialLinkText@FindBy(partialLinkText = "Forgot")
tagName@FindBy(tagName = "textarea")

@FindBys — AND Condition

All locators must match (element must satisfy every condition):

@FindBys({
    @FindBy(id = "login-form"),
    @FindBy(tagName = "input")
})
WebElement usernameInForm;

@FindAll — OR Condition

Any locator match returns the element:

@FindAll({
    @FindBy(id = "submit-btn"),
    @FindBy(css = "button[type='submit']")
})
WebElement submitButton;

initElements

public LoginPage(WebDriver driver) {
    this.driver = driver;
    PageFactory.initElements(driver, this);
}

Call in the constructor (or @BeforeMethod) to bind annotated fields to live WebElements.

@CacheLookup

@CacheLookup
@FindBy(id = "app-logo")
WebElement logo;
Warning: Use @CacheLookup only for static elements that never change. Dynamic elements cause StaleElementReferenceException.

Complete PageFactory Example

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.support.FindBy;
import org.openqa.selenium.support.PageFactory;

public class LoginPage {
    private WebDriver driver;

    @FindBy(id = "username")   private WebElement usernameField;
    @FindBy(id = "password")   private WebElement passwordField;
    @FindBy(id = "loginBtn")    private WebElement loginButton;
    @FindBy(className = "error") private WebElement errorMessage;

    public LoginPage(WebDriver driver) {
        this.driver = driver;
        PageFactory.initElements(driver, this);
    }

    public void enterUsername(String username) {
        usernameField.clear();
        usernameField.sendKeys(username);
    }

    public void enterPassword(String password) {
        passwordField.sendKeys(password);
    }

    public HomePage clickLogin() {
        loginButton.click();
        return new HomePage(driver);
    }

    public String getErrorMessage() {
        return errorMessage.getText();
    }
}

Section 29: Hybrid Automation Framework

What Is a Hybrid Framework?

A Hybrid Framework combines multiple automation approaches: POM for page structure, Data-Driven testing via Excel/JSON, Keyword-Driven reusable actions, and TestNG for execution, reporting, and parallel runs.

Framework Design Principles

  • DRY — Don't Repeat Yourself; shared utilities in one place.
  • SRP — Single Responsibility; each class has one job.
  • OCP — Open/Closed; extend via new pages/tests without modifying core.
  • Loose coupling — Tests depend on page interfaces, not driver details.
  • High cohesion — Related logic grouped (pages, utils, config).

2. ConfigReader.java

package com.framework.config;

import java.io.IOException;
import java.io.InputStream;
import java.util.Properties;

public class ConfigReader {
    private static ConfigReader instance;
    private Properties properties = new Properties();

    private ConfigReader() {
        try (InputStream is = getClass().getClassLoader()
                .getResourceAsStream("config.properties")) {
            properties.load(is);
        } catch (IOException e) {
            throw new RuntimeException("Failed to load config.properties", e);
        }
    }

    public static synchronized ConfigReader getInstance() {
        if (instance == null) instance = new ConfigReader();
        return instance;
    }

    public String get(String key) {
        return properties.getProperty(key);
    }

    public int getInt(String key) {
        return Integer.parseInt(properties.getProperty(key));
    }
}

package com.framework.base;

import com.framework.driver.DriverManager;
import com.framework.utils.WaitUtils;
import org.openqa.selenium.*;
import org.openqa.selenium.interactions.Actions;

public class BasePage {
    protected WebDriver driver;
    protected WaitUtils waitUtils;

    public BasePage() {
        this.driver = DriverManager.getDriver();
        this.waitUtils = new WaitUtils(driver);
    }

    protected void click(WebElement element) {
        waitUtils.waitForClickable(element);
        element.click();
    }

    protected void type(WebElement element, String text) {
        waitUtils.waitForVisible(element);
        element.clear();
        element.sendKeys(text);
    }

    protected void scrollTo(WebElement element) {
        new Actions(driver).scrollToElement(element).perform();
    }

    protected String getText(WebElement element) {
        waitUtils.waitForVisible(element);
        return element.getText();
    }
}

package com.framework.listeners;

import com.framework.driver.DriverManager;
import com.framework.reports.ExtentReportManager;
import com.framework.utils.ScreenshotUtils;
import com.aventstack.extentreports.ExtentTest;
import org.openqa.selenium.WebDriver;
import org.testng.ITestListener;
import org.testng.ITestResult;

public class TestListener implements ITestListener {
    @Override
    public void onTestStart(ITestResult result) {
        ExtentReportManager.createTest(result.getMethod().getMethodName());
    }

    @Override
    public void onTestFailure(ITestResult result) {
        WebDriver driver = DriverManager.getDriver();
        ExtentTest test = ExtentReportManager.getTest();
        if (driver != null) {
            String path = ScreenshotUtils.capture(driver, result.getName());
            test.addScreenCaptureFromPath(path);
            test.fail(result.getThrowable());
        }
    }

    @Override
    public void onTestSuccess(ITestResult result) {
        ExtentReportManager.getTest().pass("Test passed");
    }
}

    @Override
    public boolean retry(ITestResult result) {

        return false;

package com.framework.tests;

import com.framework.base.BaseTest;
import com.framework.dataproviders.LoginDataProvider;
import com.framework.listeners.RetryAnalyzer;
import com.framework.pages.HomePage;
import com.framework.pages.LoginPage;
import org.testng.Assert;
import org.testng.annotations.Test;
import org.testng.asserts.SoftAssert;

public class LoginTest extends BaseTest {

    @Test(dataProvider = "loginData", dataProviderClass = LoginDataProvider.class,
          groups = {"smoke"}, retryAnalyzer = RetryAnalyzer.class,
          description = "Verify login with valid and invalid credentials")
    public void testLogin(String username, String password, boolean expectedSuccess) {
        LoginPage loginPage = new LoginPage();

        if (expectedSuccess) {
            HomePage home = loginPage.login(username, password);
            SoftAssert soft = new SoftAssert();
            soft.assertTrue(home.isDashboardDisplayed(), "Dashboard not displayed");
            soft.assertEquals(home.getWelcomeText(), "Welcome, " + username + "!");
            soft.assertAll();
        } else {
            loginPage.login(username, password);
            Assert.assertTrue(loginPage.getErrorMessage().contains("Invalid"),
                "Expected error message not shown");
        }
    }

    @Test(groups = {"regression"}, dependsOnGroups = {"smoke"})
    public void testLogout() {
        new LoginPage().login("admin", "admin123");
        HomePage home = new HomePage();
        home.clickLogout();
        Assert.assertTrue(new LoginPage().isLoginPageDisplayed());
    }
