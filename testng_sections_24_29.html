Section 24: Cucumber Hooks

What Are Hooks?

Hooks in Cucumber are blocks of code that run at specific points in the test lifecycle — before or after scenarios and steps. They provide setup and teardown logic that keeps feature files clean and centralizes cross-cutting concerns like browser initialization, database seeding, logging, and screenshot capture.

Key Idea: Hooks decouple infrastructure concerns from business-readable Gherkin. Scenarios describe what to test; hooks handle how the environment is prepared and cleaned up.

Hook Annotations Overview

AnnotationScopeWhen It Runs
@BeforeScenarioBefore each scenario (after background steps)
@AfterScenarioAfter each scenario completes (pass or fail)
@BeforeStepStepBefore each step definition executes
@AfterStepStepAfter each step definition executes
@BeforeAllSuiteOnce before all scenarios (Cucumber 7+)
@AfterAllSuiteOnce after all scenarios (Cucumber 7+)

@Before — Runs Before Each Scenario

Use @Before to launch browsers, reset application state, create test data, or start logging for every scenario.

@Before
public void setUp() {
    driver = DriverFactory.getDriver();
    driver.manage().window().maximize();
    logger.info("Scenario setup complete");
}

@After — Runs After Each Scenario

Use @After for cleanup: quit browser, delete temp files, rollback DB transactions, or capture evidence on failure.

@After
public void tearDown(Scenario scenario) {
    if (scenario.isFailed()) {
        captureScreenshot(scenario);
    }
    DriverFactory.quitDriver();
}

@BeforeStep and @AfterStep

Step-level hooks run around every Gherkin step. Useful for fine-grained logging, highlighting elements before interaction, or capturing screenshots on step failure.

@BeforeStep
public void beforeStep(Scenario scenario) {
    logger.info("Starting step in scenario: " + scenario.getName());
}

@AfterStep
public void afterStep(Scenario scenario) {
    if (scenario.isFailed()) {
        byte[] screenshot = ((TakesScreenshot) driver)
            .getScreenshotAs(OutputType.BYTES);
        scenario.attach(screenshot, "image/png", "Step Failure");
    }
}

Hook with Scenario Parameter

Passing the Scenario object gives access to scenario metadata at runtime:

  • Name: scenario.getName()
  • Status: scenario.isFailed(), scenario.getStatus()
  • Tags: scenario.getSourceTagNames()
  • URI: scenario.getUri() — feature file path
  • Line: scenario.getLine() — scenario line number
@After
public void logScenarioResult(Scenario scenario) {
    logger.info("Scenario: " + scenario.getName()
        + " | Status: " + scenario.getStatus()
        + " | Tags: " + scenario.getSourceTagNames());
}

Conditional Hooks with Tags

Tag expressions in hook annotations restrict execution to matching scenarios. This avoids unnecessary setup for unrelated tests.

@Before("@login or @checkout")
public void setUpAuthContext() {
    // Only runs for login or checkout scenarios
    authToken = AuthService.getTestToken();
}

@Before("@database")
public void setUpDatabase() {
    dbHelper.connect();
    dbHelper.seedTestData();
}

@After("@database")
public void tearDownDatabase() {
    dbHelper.truncateTables();
    dbHelper.disconnect();
}

Hook Execution Order

When multiple hooks of the same type exist, use the order attribute (lower numbers run first for @Before; higher numbers run first for @After).

@Before(order = 0)
public void connectDatabase() { dbHelper.connect(); }

@Before(order = 1)
public void launchBrowser() { driver = DriverFactory.getDriver(); }

@Before(order = 2)
public void navigateToBaseUrl() { driver.get(ConfigReader.get("base.url")); }

@After(order = 2)
public void captureScreenshotOnFailure(Scenario scenario) { /* ... */ }

@After(order = 1)
public void quitBrowser() { DriverFactory.quitDriver(); }

@After(order = 0)
public void disconnectDatabase() { dbHelper.disconnect(); }
Order Rule: @Before hooks run ascending (0 → 1 → 2). @After hooks run descending (2 → 1 → 0) — mirroring a stack: last setup in, first teardown out.

Complete Hooks Class

package hooks;

import io.cucumber.java.*;
import org.openqa.selenium.*;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import utils.DriverFactory;
import utils.ConfigReader;
import utils.DatabaseHelper;

public class Hooks {

    private static final Logger logger = LoggerFactory.getLogger(Hooks.class);
    private WebDriver driver;
    private DatabaseHelper dbHelper;

    // ── One-time suite setup (Cucumber 7+) ──
    @BeforeAll
    public static void beforeAll() {
        logger.info("=== Test Suite Started ===");
        ConfigReader.load("qa");
    }

    @AfterAll
    public static void afterAll() {
        logger.info("=== Test Suite Finished ===");
    }

    // ── Scenario-level setup ──
    @Before(order = 0)
    public void startLogging(Scenario scenario) {
        logger.info("▶ Starting scenario: {} | Tags: {}",
            scenario.getName(), scenario.getSourceTagNames());
    }

    @Before(order = 1)
    public void launchBrowser() {
        driver = DriverFactory.getDriver();
        driver.manage().window().maximize();
        driver.get(ConfigReader.get("base.url"));
    }

    @Before(value = "@database", order = 0)
    public void setUpDatabase() {
        dbHelper = new DatabaseHelper();
        dbHelper.connect();
        dbHelper.insertTestUser("testuser", "pass123");
    }

    // ── Step-level hooks ──
    @BeforeStep
    public void logStep(Scenario scenario) {
        logger.debug("Executing step in: {}", scenario.getName());
    }

    @AfterStep
    public void screenshotOnStepFailure(Scenario scenario) {
        if (scenario.isFailed()) {
            attachScreenshot(scenario, "Step Failure Screenshot");
        }
    }

    // ── Scenario-level teardown ──
    @After(order = 2)
    public void captureFailureScreenshot(Scenario scenario) {
        if (scenario.isFailed()) {
            attachScreenshot(scenario, "Scenario Failure Screenshot");
            logger.error("✘ Scenario FAILED: {}", scenario.getName());
        } else {
            logger.info("✔ Scenario PASSED: {}", scenario.getName());
        }
    }

    @After(order = 1)
    public void quitBrowser() {
        DriverFactory.quitDriver();
    }

    @After(value = "@database", order = 0)
    public void cleanUpDatabase() {
        dbHelper.deleteTestUser("testuser");
        dbHelper.disconnect();
    }

    private void attachScreenshot(Scenario scenario, String name) {
        try {
            byte[] screenshot = ((TakesScreenshot) driver)
                .getScreenshotAs(OutputType.BYTES);
            scenario.attach(screenshot, "image/png", name);
        } catch (Exception e) {
            logger.warn("Could not capture screenshot: {}", e.getMessage());
        }
    }
}

@BeforeAll / @AfterAll (Cucumber 7+)

Suite-level hooks run once per test run — ideal for loading configuration, initializing report directories, or warming up connection pools. Methods must be static.

@BeforeAll
public static void globalSetup() {
    Files.createDirectories(Paths.get("target/screenshots"));
    RestAssured.baseURI = ConfigReader.get("api.base.url");
}

@AfterAll
public static void globalTeardown() {
    EmailNotifier.sendSummaryReport();
}

Hook Best Practices

  • Keep hooks thin — delegate heavy logic to utility classes; hooks should orchestrate, not implement.
  • Use tag-based conditional hooks — avoid running DB setup for UI-only scenarios.
  • Always set hook order explicitly when multiple hooks exist — implicit order is fragile.
  • Capture screenshots in @After, not @Before — browser state reflects the failure point.
  • Use @BeforeAll/@AfterAll sparingly — static state can cause issues in parallel runs.
  • Never put assertions in hooks — hooks prepare/teardown; steps validate behavior.
  • Make hooks idempotent — safe to run even if a prior step partially succeeded.
  • Log scenario name and tags — essential for debugging parallel and tagged runs.

// Custom listener for retry and reporting
public class RetryListener implements IAnnotationTransformer {
    @Override
    public void transform(ITestAnnotation annotation,
                          Class testClass,
                          Constructor testConstructor,
                          Method testMethod) {
        annotation.setRetryAnalyzer(RetryAnalyzer.class);
    }
}

// RetryAnalyzer.java
public class RetryAnalyzer implements IRetryAnalyzer {
    private int count = 0;
    private static final int MAX_RETRY = 2;

    @Override
    public boolean retry(ITestResult result) {
        if (count < MAX_RETRY) {
            count++;
            return true;
        }
        return false;
    }
}

2. DriverFactory.java — ThreadLocal Browser Management

package utils;

import io.github.bonigarcia.wdm.WebDriverManager;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.chrome.ChromeOptions;
import org.openqa.selenium.firefox.FirefoxDriver;
import org.openqa.selenium.edge.EdgeDriver;

public class DriverFactory {

    private static final ThreadLocal<WebDriver> driverThread = new ThreadLocal<>();

    public static WebDriver getDriver() {
        if (driverThread.get() == null) {
            driverThread.set(createDriver());
        }
        return driverThread.get();
    }

    private static WebDriver createDriver() {
        String browser = System.getProperty("browser", "chrome").toLowerCase();
        boolean headless = Boolean.parseBoolean(
            System.getProperty("headless", "false"));

        return switch (browser) {
            case "firefox" -> {
                WebDriverManager.firefoxdriver().setup();
                yield new FirefoxDriver();
            }
            case "edge" -> {
                WebDriverManager.edgedriver().setup();
                yield new EdgeDriver();
            }
            default -> {
                WebDriverManager.chromedriver().setup();
                ChromeOptions options = new ChromeOptions();
                if (headless) options.addArguments("--headless=new");
                options.addArguments("--disable-gpu", "--no-sandbox");
                yield new ChromeDriver(options);
            }
        };
    }

    public static void quitDriver() {
        if (driverThread.get() != null) {
            driverThread.get().quit();
            driverThread.remove();
        }
    }
}

package utils;

import java.io.IOException;
import java.io.InputStream;
import java.util.Properties;

public class ConfigReader {

    private static Properties properties = new Properties();

    public static void load(String env) {
        String file = "config/" + env + ".properties";
        try (InputStream input = ConfigReader.class
                .getClassLoader().getResourceAsStream(file)) {
            if (input == null) {
                throw new RuntimeException("Config file not found: " + file);
            }
            properties.load(input);
        } catch (IOException e) {
            throw new RuntimeException("Failed to load config: " + file, e);
        }
    }

    public static String get(String key) {
        String sysVal = System.getProperty(key);
        return sysVal != null ? sysVal : properties.getProperty(key);
    }
}

4. BasePage.java — Common Page Methods

package pages;

import org.openqa.selenium.*;
import org.openqa.selenium.support.PageFactory;
import org.openqa.selenium.support.ui.ExpectedConditions;
import org.openqa.selenium.support.ui.WebDriverWait;
import utils.DriverFactory;
import java.time.Duration;

public abstract class BasePage {

    protected WebDriver driver;
    protected WebDriverWait wait;

    public BasePage() {
        this.driver = DriverFactory.getDriver();
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(15));
        PageFactory.initElements(driver, this);
    }

    protected void click(WebElement element) {
        wait.until(ExpectedConditions.elementToBeClickable(element)).click();
    }

    protected void type(WebElement element, String text) {
        wait.until(ExpectedConditions.visibilityOf(element));
        element.clear();
        element.sendKeys(text);
    }

    protected String getText(WebElement element) {
        return wait.until(ExpectedConditions.visibilityOf(element)).getText();
    }

    protected boolean isDisplayed(WebElement element) {
        try {
            return wait.until(ExpectedConditions.visibilityOf(element)).isDisplayed();
        } catch (TimeoutException e) {
            return false;
        }
    }
}

Complete APISteps.java with REST Assured

package stepdefinitions;

import context.TestContext;
import io.cucumber.datatable.DataTable;
import io.cucumber.java.en.*;
import io.restassured.response.Response;
import io.restassured.specification.RequestSpecification;
import org.testng.Assert;

import java.util.List;
import java.util.Map;

import static io.restassured.RestAssured.*;
import static org.hamcrest.Matchers.*;

public class APISteps {

    private TestContext testContext;
    private RequestSpecification request;
    private Response response;

    public APISteps(TestContext testContext) {
        this.testContext = testContext;
    }

    @Given("the API base URL is {string}")
    public void setBaseUrl(String baseUrl) {
        baseURI = baseUrl;
        request = given()
            .header("Content-Type", "application/json")
            .header("Accept", "application/json");
    }

    @Given("a user exists with id {string}")
    public void userExists(String id) {
        testContext.set("userId", id);
    }

    @When("I send a GET request to {string}")
    public void sendGetRequest(String endpoint) {
        response = request.when().get(endpoint);
        testContext.set("response", response);
    }

    @When("I send a POST request to {string} with body:")
    public void sendPostRequest(String endpoint, DataTable dataTable) {
        Map<String, String> body = dataTable.asMaps().get(0);
        response = request
            .body(body)
            .when()
            .post(endpoint);
        testContext.set("response", response);
        testContext.set("requestBody", body);
    }

    @When("I send a PUT request to {string} with body:")
    public void sendPutRequest(String endpoint, DataTable dataTable) {
        Map<String, String> body = dataTable.asMaps().get(0);
        response = request.body(body).when().put(endpoint);
        testContext.set("response", response);
    }

    @When("I send a DELETE request to {string}")
    public void sendDeleteRequest(String endpoint) {
        response = request.when().delete(endpoint);
        testContext.set("response", response);
    }

    @Then("the response status code should be {int}")
    public void verifyStatusCode(int expectedStatus) {
        response = (Response) testContext.get("response");
        Assert.assertEquals(response.getStatusCode(), expectedStatus,
            "Status code mismatch. Body: " + response.getBody().asString());
    }

    @Then("the response body should contain {string}")
    public void verifyBodyContains(String expectedText) {
        response = (Response) testContext.get("response");
        Assert.assertTrue(response.getBody().asString().contains(expectedText),
            "Response body does not contain: " + expectedText);
    }

    @Then("the response JSON path {string} should be {string}")
    public void verifyJsonPath(String jsonPath, String expectedValue) {
        response = (Response) testContext.get("response");
        String actual = response.jsonPath().getString(jsonPath);
        Assert.assertEquals(actual, expectedValue,
            "JSONPath " + jsonPath + " mismatch");
    }

    @Then("the response header {string} should be {string}")
    public void verifyHeader(String headerName, String expectedValue) {
        response = (Response) testContext.get("response");
        Assert.assertEquals(response.getHeader(headerName), expectedValue);
    }

    @Then("the response time should be less than {int} ms")
    public void verifyResponseTime(int maxMs) {
        response = (Response) testContext.get("response");
        Assert.assertTrue(response.getTime() < maxMs,
            "Response time " + response.getTime() + "ms exceeded " + maxMs + "ms");
    }
}

Jenkins orchestrates the full BDD test lifecycle — from code checkout through Maven build, parallel TestNG/Cucumber execution, report generation, and stakeholder notification.

Complete Jenkinsfile

pipeline {
    agent any

    parameters {
        choice(name: 'TAG', choices: ['@smoke', '@regression', '@full'],
               description: 'Cucumber tag to execute')
        choice(name: 'ENV', choices: ['qa', 'staging'],
               description: 'Target environment')
        choice(name: 'BROWSER', choices: ['chrome', 'firefox', 'edge'],
               description: 'Browser for UI tests')
        booleanParam(name: 'HEADLESS', defaultValue: true,
                     description: 'Run browser in headless mode')
    }

    environment {
        MAVEN_OPTS = '-Xmx1024m'
        ALLURE_RESULTS = 'target/allure-results'
    }

    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }

        stage('Build') {
            steps {
                sh 'mvn clean compile -B'
            }
        }

        stage('Smoke Tests') {
            when { expression { params.TAG == '@smoke' || params.TAG == '@full' } }
            steps {
                sh """
                    mvn test -B \
                      -Dcucumber.filter.tags="@smoke" \
                      -Denv=${params.ENV} \
                      -Dbrowser=${params.BROWSER} \
                      -Dheadless=${params.HEADLESS}
                """
            }
            post {
                always {
                    cucumber fileIncludePattern: 'target/cucumber.json'
                    archiveArtifacts artifacts: 'target/cucumber-report.html', allowEmptyArchive: true
                }
            }
        }

        stage('Regression Tests') {
            when { expression { params.TAG == '@regression' || params.TAG == '@full' } }
            parallel {
                stage('Regression - Chrome') {
                    steps {
                        sh """
                            mvn test -B \
                              -Dcucumber.filter.tags="@regression" \
                              -Denv=${params.ENV} \
                              -Dbrowser=chrome \
                              -Dheadless=true
                        """
                    }
                }
                stage('Regression - Firefox') {
                    steps {
                        sh """
                            mvn test -B \
                              -Dcucumber.filter.tags="@regression" \
                              -Denv=${params.ENV} \
                              -Dbrowser=firefox \
                              -Dheadless=true
                        """
                    }
                }
            }
            post {
                always {
                    cucumber fileIncludePattern: 'target/cucumber.json'
                }
            }
        }

        stage('Allure Report') {
            steps {
                allure includeProperties: false,
                       jdk: '',
                       results: [[path: "${ALLURE_RESULTS}"]]
            }
        }
    }

    post {
        always {
            junit 'target/cucumber-junit.xml'
            archiveArtifacts artifacts: 'target/**/*-report*, target/allure-results/**',
                             allowEmptyArchive: true
        }
        success {
            emailext(
                subject: "✔ BDD Tests PASSED - ${env.JOB_NAME} #${env.BUILD_NUMBER}",
                body: """Build ${env.BUILD_URL} succeeded.
                         Environment: ${params.ENV} | Tag: ${params.TAG}
                         Allure Report: ${env.BUILD_URL}allure""",
                to: '${DEFAULT_RECIPIENTS}'
            )
        }
        failure {
            emailext(
                subject: "✘ BDD Tests FAILED - ${env.JOB_NAME} #${env.BUILD_NUMBER}",
                body: """Build ${env.BUILD_URL} failed.
                         Check console output and Allure report.""",
                to: '${DEFAULT_RECIPIENTS}'
            )
        }
    }
}

HEADLESStrue / falseHeadless mode for CI agents without display

Maven Command in Jenkins

# Smoke tests on QA with Chrome
mvn clean test -Dcucumber.filter.tags="@smoke" -Dbrowser=chrome -Denv=qa

# Regression with parallel threads
mvn clean test -Dcucumber.filter.tags="@regression" -Dbrowser=chrome -Denv=staging -Dheadless=true

# API tests only
mvn clean test -Dcucumber.filter.tags="@api" -Denv=qa

# Full suite with Allure
mvn clean test -Denv=qa -Dbrowser=chrome
allure generate target/allure-results -o target/allure-report

Cucumber JSON Report in Jenkins

Install the Cucumber Reports Plugin in Jenkins. After test execution, publish results:

post {
    always {
        cucumber fileIncludePattern: '**/cucumber.json'
        archiveArtifacts artifacts: 'target/cucumber-report.html'
    }
}

This renders feature/scenario pass-fail trends directly in the Jenkins build page with drill-down to failed steps.

Allure Report in Jenkins

Install the Allure Plugin. Configure the pipeline stage:

stage('Allure Report') {
    steps {
        allure includeProperties: false,
               results: [[path: 'target/allure-results']]
    }
}

Allure dashboard appears as a sidebar link on each build with history, severity, and screenshot attachments.

triggers {

Slack / Teams Notification

post {
    always {
        script {
            def status = currentBuild.currentResult
            def color  = status == 'SUCCESS' ? 'good' : 'danger'
            def emoji  = status == 'SUCCESS' ? '✔' : '✘'

            slackSend(
                channel: '#qa-automation',
                color: color,
                message: "${emoji} *${env.JOB_NAME}* #${env.BUILD_NUMBER} - ${status}\n" +
                         "Tag: ${params.TAG} | Env: ${params.ENV} | Browser: ${params.BROWSER}\n" +
                         "<${env.BUILD_URL}|View Build> | <${env.BUILD_URL}allure|Allure Report>"
            )
        }
    }
}
Microsoft Teams: Use the Office 365 Connector plugin or webhook notification step with Adaptive Card JSON containing build status, report links, and failure summary.

Jenkins Pipeline Best Practices for BDD

  • Archive reports always — use post { always { } } so reports are saved even on failure.
  • Parameterize everything — tag, env, browser — avoid hardcoding in Jenkinsfile.
  • Run smoke on every commit — gate merges with fast @smoke suite (< 10 min).
  • Parallel regression — split by browser or feature tag in parallel stages.
  • Headless by default — CI agents rarely have a display; pass -Dheadless=true.
  • Separate API and UI stages — API tests run faster without browser overhead.
  • Pin Maven/ JDK versions — use Jenkins tools directive for reproducible builds.
