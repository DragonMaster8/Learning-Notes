Module 1: UNIX/Linux Fundamentals

What is UNIX?

UNIX is a multi-user, multi-tasking operating system developed in 1969 at AT&T Bell Labs by Ken Thompson and Dennis Ritchie. It's the foundation of modern operating systems and introduced concepts like the file system hierarchy, pipes, and the shell.

What is Linux?

Linux is a free, open-source UNIX-like operating system created by Linus Torvalds in 1991. It's the kernel (core) that powers most servers, CI/CD pipelines, cloud infrastructure, and containerized applications used in testing.

Key Components Explained

Kernel

The core of Linux that interfaces directly with hardware. QA engineers rarely interact with the kernel directly, but understanding it helps debug performance issues (CPU scheduling, memory allocation, I/O bottlenecks).

Shell

The command-line interpreter that executes your commands. Bash (Bourne Again Shell) is the default on most Linux systems. Every command you type in terminal goes through the shell.

# Check your current shell
echo $SHELL          # Output: /bin/bash

# Available shells on the system
cat /etc/shells

Terminal

The interface (window) where you type shell commands. It's just a text-based UI to the shell. Tools: Terminal (Ubuntu), iTerm2 (Mac), PuTTY/MobaXterm (Windows SSH clients).

Understanding ls -l Output

$ ls -l

drwxr-xr-x 5 tester qa-team 4096 Jul 10 09:15 src/

# Breakdown:
# -rw-r--r--  → File type + permissions (- = file, d = directory)

Common Mistakes

  • rm -rf / — NEVER run this. Deletes entire system. Always double-check paths before rm -rf
  • Running rm -rf in wrong directory — always pwd first
  • Forgetting -r flag when copying directories — copies nothing silently
  • Using spaces in file/directory names without quotes — cd "my folder" not cd my folder
  • Not using -p with mkdir for nested paths — fails if parent doesn't exist

Interview Questions

  1. What is the difference between rm and rmdir?
    rmdir only removes EMPTY directories. rm -r removes directories with all contents recursively.
  2. How do you find the most recently modified file in a directory?
    ls -ltr — sorts by time with newest at bottom. Or ls -lt | head -5 for top 5 newest.
  3. What does ls -la show that ls doesn't?
    Hidden files (starting with .), detailed permissions, owner, group, size, and timestamps.

# Space/f     = page forward
# b           = page backward

# N           = previous match

# g           = go to beginning

grep — The QA Engineer's Best Friend

# Basic search
grep "ERROR" application.log                    # Lines containing ERROR
grep -i "error" application.log                 # Case-insensitive
grep -n "ERROR" application.log                 # With line numbers
grep -c "ERROR" application.log                 # Count of matching lines

# Context (surrounding lines)
grep -A 5 "ERROR" app.log                       # 5 lines AFTER match
grep -B 3 "ERROR" app.log                       # 3 lines BEFORE match
grep -C 3 "ERROR" app.log                       # 3 lines BEFORE + AFTER

# Recursive search in directory
grep -r "NullPointerException" /var/log/        # Search all files in dir
grep -rl "TODO" tests/                          # List filenames only

# Multiple patterns
grep -E "ERROR|WARN|FATAL" application.log      # OR patterns (egrep)
grep "ERROR" app.log | grep -v "expected"       # Exclude pattern (-v = invert)

# Regular expressions
grep "2026-07-10.*ERROR" app.log                # Date-specific errors
grep "response.*[45][0-9][0-9]" api.log         # 4xx/5xx status codes

Real-Time QA Scenarios

Scenario 1: Application Failed After Deployment — Investigate

# Step 1: Check recent logs
tail -100 /var/log/application/app.log

# Step 2: Find errors since deployment time

grep -A 20 "Exception" app.log | tail -50

# Step 4: Monitor live for new errors
tail -f app.log | grep --color "ERROR\|Exception"

# Step 5: Count errors per hour
grep "ERROR" app.log | awk '{print $1, $2}' | cut -d: -f1,2 | sort | uniq -c

Common Mistakes

  • Using cat on huge log files (500MB+) — freezes terminal. Use less or tail
  • Forgetting -i flag — searching "error" misses "ERROR" and "Error"
  • Not using | grep -v to filter noise from results
  • Piping tail -f through too many greps — performance issues

Interview Questions

  1. How do you monitor a log file in real-time?
    tail -f /path/to/logfile.log — follows the file and shows new lines as they are appended.
  2. How to find all ERROR lines in the last hour of a log?
    grep "$(date +%Y-%m-%d\ %H)" app.log | grep "ERROR" or use awk for time range filtering.
  3. Difference between grep -r and grep -l?
    -r = recursive search in all files. -l = show only filenames (not the matching lines).
  4. How to count the number of errors in a log file?
    grep -c "ERROR" app.log — returns count of matching lines.

Module 5: File Permissions Frequently Asked

Concept Explanation

Every file and directory in Linux has three types of permissions for three categories of users. Understanding permissions is critical for QA engineers because permission issues are one of the most common causes of automation failures.

Permission Structure

-rwxr-xr-- 1 tester qa-team 4096 Jul 10 08:30 run_tests.sh │└┬┘└┬┘└┬┘ │ │ │ └── Others (everyone else): r-- (read only) │ │ └────── Group (qa-team): r-x (read + execute) │ └────────── Owner (tester): rwx (read + write + execute) └──────────── File type: - (regular file), d (directory)

Permission Types

PermissionSymbolNumericFile MeaningDirectory Meaning
Readr4View file contentsList directory contents (ls)
Writew2Modify file contentsCreate/delete files in directory
Executex1Run as program/scriptEnter directory (cd)
No permission-0Cannot accessCannot access

Common Permission Numbers

NumberPermissionUse Case
777rwxrwxrwx (everyone can do everything)DANGEROUS — never use in production
755rwxr-xr-x (owner full, others read+execute)Scripts, executables, directories
644rw-r--r-- (owner read/write, others read only)Config files, data files
600rw------- (owner only)SSH keys, secrets, .env files
700rwx------ (owner only, executable)Private scripts

chmod — Change Permissions

# Numeric mode
chmod 755 run_tests.sh        # Owner: rwx, Group: r-x, Others: r-x
chmod 644 config.properties   # Owner: rw-, Group: r--, Others: r--
chmod 600 .env                # Owner only: rw- (secrets!)

# Symbolic mode
chmod +x script.sh            # Add execute permission for all
chmod u+x script.sh           # Add execute for owner only
chmod g+w file.txt            # Add write for group
chmod o-r file.txt            # Remove read from others
chmod u+rwx,g+rx,o+r file    # Full symbolic specification

# Recursive (all files in directory)
chmod -R 755 /opt/automation/

chown & chgrp — Change Ownership

# Change owner
sudo chown tester run_tests.sh
sudo chown jenkins:jenkins /var/lib/jenkins/workspace/

# Change owner and group together
sudo chown tester:qa-team project/

# Change group only
sudo chgrp qa-team test-reports/

# Recursive ownership change
sudo chown -R jenkins:jenkins /opt/automation/

QA Troubleshooting Scenarios

Scenario 1: Automation Script Failed — Permission Denied

# Error: bash: ./run_tests.sh: Permission denied

# Check permissions
ls -l run_tests.sh
# -rw-r--r-- 1 tester qa-team 1024 Jul 10 run_tests.sh
# Problem: No execute permission (missing x)

# Fix:
chmod +x run_tests.sh
# Or: chmod 755 run_tests.sh

Scenario 2: Jenkins Unable to Execute Scripts

# Error in Jenkins console: Permission denied

# Check who Jenkins runs as
ps -ef | grep jenkins    # Usually runs as 'jenkins' user

# Check file ownership
ls -la /opt/automation/run_tests.sh
# Owner: tester (not jenkins!)

# Fix: Change ownership OR add jenkins to group
sudo chown jenkins:jenkins /opt/automation/run_tests.sh
# Or: sudo chmod 755 /opt/automation/run_tests.sh

Scenario 3: SSH Key Permission Error

# Error: Permissions 0644 for 'id_rsa' are too open. Key will be ignored.

# Fix: SSH keys MUST be 600 (owner read/write only)
chmod 600 ~/.ssh/id_rsa
chmod 644 ~/.ssh/id_rsa.pub   # Public key can be readable

Interview Questions

  1. What does chmod 755 mean?
    Owner: read+write+execute (7). Group: read+execute (5). Others: read+execute (5). Common for scripts and executables.
  2. Your automation script fails with "Permission denied." How do you fix it?
    Check with ls -l script.sh. Add execute: chmod +x script.sh. Verify user running it has access.
  3. What's the difference between chmod and chown?
    chmod changes permissions (what can be done). chown changes ownership (who owns the file).
  4. Why should SSH keys have 600 permission?
    SSH refuses keys that are readable by others (security). Only the owner should read/write private keys.
Quick Revision — Module 5 (12 items)
  1. Three permissions: Read (4), Write (2), Execute (1)
  2. Three user categories: Owner (u), Group (g), Others (o)
  3. chmod 755 = rwxr-xr-x (scripts/executables)
  4. chmod 644 = rw-r--r-- (config/data files)
  5. chmod 600 = rw------- (secrets, SSH keys)
  6. chmod +x = add execute permission
  7. chown user:group file = change ownership
  8. "Permission denied" = most common QA automation failure
  9. Jenkins runs as 'jenkins' user — files must be accessible
  10. Always ls -l to diagnose permission issues
  11. SSH keys: private = 600, public = 644
  12. Never use 777 in production — security vulnerability

Module 13: Log Analysis for Testers Frequently Asked

Why Log Analysis is Critical for QA

As a QA engineer, you'll investigate application failures, slow APIs, and production issues daily. Log analysis using Linux commands is often faster and more efficient than using any GUI tool.

Log Analysis Workflow

Issue Reported │ ▼ 1. Identify log location (find, locate) │ ▼ 2. Check recent entries (tail -100, tail -f) │ ▼ 3. Filter for errors (grep ERROR, grep Exception) │ ▼ 4. Get context (grep -B5 -A10 "pattern") │ ▼ 5. Analyze patterns (awk, sort, uniq -c) │ ▼ 6. Correlate timestamps (grep "timerange") │ ▼ Root Cause Identified → Report to Dev Team

Scenario 1: Production Login Failures

# Step 1: Find authentication logs
find /var/log -name "*auth*" -o -name "*login*" | head

# Step 2: Check recent login failures
tail -500 /var/log/app/auth.log | grep -i "failed\|denied\|invalid"

# Step 3: Count failures by user
grep "login failed" auth.log | awk '{print $NF}' | sort | uniq -c | sort -rn | head -10
# Output:
#   145  user@email.com
#    89  admin@company.com
#    23  test@example.com

# Step 4: Check if it's from specific IP (potential brute force)
grep "login failed" auth.log | awk '{print $5}' | sort | uniq -c | sort -rn
# If one IP has thousands → possible attack

# Step 5: Get timeline of failures
grep "login failed" auth.log | awk '{print $1, $2}' | cut -d: -f1,2 | uniq -c
# Shows failures per hour

Scenario 2: Slow API Response Investigation

# Step 1: Find API response time entries
grep "response_time" api-gateway.log | tail -100

# Step 2: Extract slow requests (>5000ms)
awk '/response_time/ && $NF > 5000 {print $0}' api-gateway.log

# Step 3: Identify slowest endpoints
grep "response_time" api.log | awk '{print $(NF-2), $NF}' | sort -k2 -n | tail -20

# Step 4: Average response time per endpoint
awk -F'|' '/GET/ {sum[$3]+=$NF; count[$3]++} END {for(e in sum) print e, sum[e]/count[e] "ms"}' api.log

# Step 5: Check if DB is the bottleneck
grep "db_query_time" app.log | awk '{if($NF > 1000) print $0}' | wc -l

# Step 6: Real-time monitoring of slow requests
tail -f api.log | awk '/response_time/ && $NF > 3000 {print "\033[31m" $0 "\033[0m"}'

Scenario 3: Database Connection Failures

# Find DB connection errors
grep -i "connection\|timeout\|refused\|pool" app.log | grep -i "error\|fail\|exception"

# Check connection pool exhaustion
grep "pool" app.log | grep -i "exhaust\|full\|wait" | tail -20

# Count DB errors over time
grep "DBConnectionException" app.log | awk '{print $1, $2}' | cut -d: -f1 | uniq -c

# Check if it correlates with high traffic
grep "active_connections" app.log | awk '{print $1, $NF}' | tail -50

Scenario 4: Server Crash / Out of Memory

# Check system logs for OOM (Out of Memory) killer
sudo grep -i "oom\|out of memory\|killed process" /var/log/syslog
sudo dmesg | grep -i "oom\|memory"

# Check application heap dumps
find /var/log -name "*.hprof" -mtime -1

# Memory usage over time
grep "heap\|memory" app.log | awk '/used/ {print $1, $2, $NF}'

# Check disk space (log files filling disk)
df -h                          # Filesystem usage
du -sh /var/log/*              # Size of each log directory
find /var/log -size +1G        # Logs over 1GB

Scenario 5: Automation Job Failed at Midnight

# Check Jenkins build log
cat /var/lib/jenkins/jobs/nightly-regression/builds/lastFailedBuild/log | tail -100

# Find the exact failure point
grep -n "BUILD FAILURE\|FATAL\|Exception" build.log | head -5

# Check if it was an environment issue
grep -i "connection refused\|timeout\|unavailable" build.log

# Check if server was under load at that time
sar -u -s 00:00:00 -e 01:00:00    # CPU usage midnight to 1AM
sar -r -s 00:00:00 -e 01:00:00    # Memory usage

# Check cron jobs that might have conflicted
grep "CRON" /var/log/syslog | grep "$(date -d yesterday +%b\ %d)"

Power Commands for Log Analysis

# Multi-line log entries (stack traces)
grep -A 30 "NullPointerException" app.log | head -50

# Extract unique exception types
grep "Exception" app.log | sed 's/.*\(.*Exception\).*/\1/' | sort | uniq -c | sort -rn

# Time-based filtering (logs between 2PM-3PM)
awk '$2 >= "14:00:00" && $2 <= "15:00:00"' app.log

# Compare today's errors vs yesterday
today=$(grep -c "ERROR" app_$(date +%Y-%m-%d).log)
yesterday=$(grep -c "ERROR" app_$(date -d yesterday +%Y-%m-%d).log)
echo "Today: $today errors | Yesterday: $yesterday errors"

# Error rate calculation
total=$(wc -l < access.log)
errors=$(grep -c " [45][0-9][0-9] " access.log)
echo "Error rate: $(echo "scale=4; $errors/$total*100" | bc)%"

Loops

#!/bin/bash
# Check multiple servers before test execution

SERVERS=("app-server-1" "app-server-2" "db-server" "redis-server")

echo "=== Pre-Test Health Check ==="
for server in "${SERVERS[@]}"; do
    if ping -c 1 -W 2 "$server" > /dev/null 2>&1; then
        echo "✅ $server — Reachable"
    else
        echo "❌ $server — UNREACHABLE"
        exit 1
    fi
done
echo "All servers healthy. Proceeding with tests."

# While loop — wait for service to be ready
echo "Waiting for application to start..."
RETRIES=30
while [ $RETRIES -gt 0 ]; do
    if curl -s http://localhost:8080/health | grep -q "UP"; then
        echo "✅ Application is UP"
        break
    fi
    echo "Waiting... ($RETRIES retries left)"
    sleep 5
    RETRIES=$((RETRIES - 1))
done

if [ $RETRIES -eq 0 ]; then
    echo "❌ Application failed to start within timeout"
    exit 1
fi

curl — The QA Swiss Army Knife

HTTP Methods

API Health Check Script

#!/bin/bash
# api_healthcheck.sh — Check multiple endpoints

ENDPOINTS=(
    "https://api.myapp.com/health|Health"
    "https://api.myapp.com/users|Users API"
    "https://api.myapp.com/orders|Orders API"
)

echo "=== API Health Check $(date) ==="
ALL_HEALTHY=true

for entry in "${ENDPOINTS[@]}"; do
    IFS='|' read -r url name <<< "$entry"
    status=$(curl -s -o /dev/null -w "%{http_code}" -m 10 "$url")
    if [ "$status" -eq 200 ]; then
        echo "✅ $name — HTTP $status"
    else
        echo "❌ $name — HTTP $status"
        ALL_HEALTHY=false
    fi
done

if [ "$ALL_HEALTHY" = true ]; then
    echo "All APIs healthy!"
    exit 0
else
    echo "Some APIs are DOWN!"
    exit 1
fi
